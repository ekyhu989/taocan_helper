## 📌 项目简介

本项目是一套**标准化全流程闭环AI执行系统**，采用多角色Agent隔离协作模式，以「需求不漂移、执行不混乱、排查不猜测、结果可追溯、系统可进化」为核心原则。

### 核心特性

- ✅ **8环节流程控制**：需求→设计→方案→计划→执行→审查→修复→发布，严格顺序流转
- ✅ **多角色Agent隔离**：不同角色使用不同AI模型（PM用Qwen3.6，后端用GLM5.1，前端用DeepSeek）
- ✅ **防幻觉机制**：API使用前三步验证，禁止经验假设
- ✅ **问题排查四步法**：收集证据→分析模式→提出假设→验证修复
- ✅ **4层自动进化系统**：静默记录→高频固化→专项优化→新增技能
- ✅ **虚拟Hooks机制**：提交前校验、任务停止闸门、反馈捕获

## 🛡️ 防幻觉机制

### 规则1：API使用前三步验证

```
第1步：read_file查看模块源码
第2步：search_codebase搜索使用示例
第3步：确认API签名后再调用
```

### 规则2：禁止经验假设

- ❌ "我猜这个API应该存在"
- ✅ "我已read_file验证该API存在"

### 规则3：不确定时提问

遇到不确定的地方，必须询问用户，不得自行编造

## 🚨 违规处理

### 严重违规（必须回滚）

1. **AI幻觉**：凭经验编造API、捏造规范
2. **不验证就提交**：代码/设计/文档未自我审查就报告完成
3. **数据丢失**：数据库操作未备份，导致数据丢失

### 一般违规（警告并重做）

1. **范围蔓延**：超出任务范围修改无关代码
2. **规范违反**：使用硬编码颜色、相对路径require
3. **不记录**：解决问题后不更新工作记忆

### 处理流程

```
第1次违规：警告并要求重新执行
第2次违规：回滚所有修改
第3次违规：记录到项目审查报告，PM介入
```

## 📝 Git规范

### Commit格式

```
type: 描述 (#任务ID)

type可选值：
- feat: 新功能
- fix: Bug修复
- docs: 文档更新
- style: 代码格式（不影响功能）
- refactor: 重构
- perf: 性能优化
- test: 测试
- chore: 构建过程或辅助工具的变动
```

### 示例

```
✅ feat: 添加用户登录功能 (#T1.0-1)
✅ fix: 修复支付回调签名验证 (#T1.0-2)
❌ fix: 修复bug（缺少任务ID）
```

## 💡 最佳实践
