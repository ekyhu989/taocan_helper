# Hook: feedback-detect（反馈捕获）

> **触发时机**：用户反馈、优化建议产生时  
> **执行角色**：反馈观测子Agent  
> **优先级**：高

---

## 触发条件

- 用户提出修改意见时
- 用户提出优化建议时
- 审查发现新问题时
- 任务执行遇到异常时

---

## 捕获内容

### 1. 反馈基本信息

- **反馈来源**：用户/审查子Agent/系统检测
- **反馈时间**：YYYY-MM-DD HH:mm:ss
- **关联任务**：任务ID或名称
- **当前环节**：需求/设计/方案/计划/执行/审查/修复/发布

### 2. 反馈内容分类

| 类型 | 说明 | 示例 |
|------|------|------|
| Bug反馈 | 功能异常、错误提示 | "支付回调失败，签名验证不通过" |
| 优化建议 | 性能、体验、代码质量 | "数据库查询可以加索引优化" |
| 需求变更 | 需求调整、范围扩展 | "增加微信支付渠道" |
| 流程问题 | 环节卡点、规则不合理 | "审查环节耗时太长" |

### 3. 反馈处理状态

- **待处理**：刚捕获，未分配
- **处理中**：已分配给对应角色
- **已解决**：问题已修复/建议已采纳
- **已拒绝**：不采纳（需说明原因）

---

## 执行动作

### 捕获流程

```
1. 检测反馈产生
2. 自动提取反馈内容
3. 分类打标（Bug/优化/需求/流程）
4. 生成反馈记录
5. 同步归档到 .workbuddy/knowledge/evolution-data.md
6. 通知主控制Agent
```

### 记录格式

```markdown
## 反馈记录 #F001

**时间**：2026-05-01 14:30
**来源**：用户
**任务**：支付回调功能开发
**环节**：执行
**类型**：Bug反馈
**内容**：支付回调失败，签名验证不通过
**处理状态**：待处理
**关联进化**：待评估（如出现≥3次则触发规则固化）
```

---

## 联动进化系统

### 触发第二层进化（高频反馈规则固化）

- **条件**：同一类反馈出现≥3次
- **动作**：自动提取共性规律，生成优化规则补丁
- **示例**：API幻觉反馈3次 → 强化hallucination-prevention技能

### 触发第三层进化（低效技能专项优化）

- **条件**：某技能执行通过率<80%
- **动作**：分析低效根源，生成专项优化方案
- **示例**：code-quality检查清单执行率60% → 精简为4类核心检查点

### 触发第四层进化（新增能力技能生成）

- **条件**：高频操作无对应技能覆盖
- **动作**：归纳操作逻辑，生成新技能
- **示例**：问题排查出现5次 → 生成issue-fixer.md（标准四步法）
