# Hook: stop-gate（任务停止闸门）

> **触发时机**：任务完成准备停止前  
> **执行角色**：主控制Agent  
> **优先级**：最高

---

## 触发条件

- 子Agent准备终止任务时
- 任务标记为"完成"时
- 准备进入下一环节前

---

## 校验清单

### 1. 审查通过验证

- [ ] 当前环节审查报告已生成
- [ ] 审查结果为"通过"
- [ ] 审查报告已归档到docs目录

### 2. 问题修复验证

- [ ] 审查发现的所有问题已修复
- [ ] 回归验证已通过
- [ ] 修复记录已归档

### 3. 交付物归档验证

- [ ] 所有交付物已生成
- [ ] 交付物已归档到正确目录
- [ ] 交付物命名符合规范

### 4. 文档同步验证

- [ ] MEMORY.md已更新
- [ ] CHANGELOG.md已记录
- [ ] 工作记忆已保存

---

## 执行动作

### 校验通过

```
✅ 所有闸门检查通过
→ 允许终止当前任务
→ 关闭子Agent实例
→ 进入下一环节或标记任务完成
```

### 校验不通过

```
❌ 闸门检查未通过
→ 禁止终止任务
→ 列出缺失项
→ 要求子Agent继续执行
```

---

## 强制规则

- **未通过闸门检查，绝对禁止停止任务**
- **子Agent不得擅自关闭，必须通过主控制Agent验证**
- **所有检查项必须100%通过，不允许部分通过**

---

*本Hook为任务终止的唯一闸门，不可绕过。*
