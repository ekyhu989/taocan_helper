# Hook: pre-commit-check（提交前校验）

> **触发时机**：所有文件git commit前  
> **执行角色**：审查子Agent  
> **优先级**：最高

---

## 触发条件

- 执行 `git commit` 命令前
- 执行 `git push` 命令前
- 任务完成准备提交时

---

## 校验清单

### 1. 格式合规检查

- [ ] Markdown文件遵循标准格式（标题层级、列表格式、代码块）
- [ ] JSON文件格式正确（无语法错误）
- [ ] 代码文件遵循项目编码规范

### 2. 命名规范检查

- [ ] 文件名符合规范（kebab-case或camelCase）
- [ ] 目录归属正确（.agent/、.workbuddy/、docs/等）
- [ ] 版本号格式正确（MAJOR.MINOR.PATCH）

### 3. 审查通过检查

- [ ] 当前环节审查报告已生成
- [ ] 审查结果为"通过"
- [ ] 所有问题已修复或标注为"接受风险"

### 4. 文档完整性检查

- [ ] 需求文档已更新（如涉及需求变更）
- [ ] 设计文档已更新（如涉及设计变更）
- [ ] CHANGELOG.md已记录变更
- [ ] VERSION.md已更新版本号

---

## 执行动作

### 校验通过

```
✅ 所有检查项通过
→ 允许提交
→ 记录提交日志到 .workbuddy/memory/MEMORY.md
```

### 校验不通过

```
❌ 发现N个不合规项
→ 拦截提交
→ 生成整改清单
→ 要求修复后重新提交
```

---

## 违规处理

- **第1次**：警告并列出所有不合规项
- **第2次**：回滚所有未提交的修改
- **第3次**：记录到项目审查报告，PM介入

---

*本Hook为全局强制校验，不可跳过。*
