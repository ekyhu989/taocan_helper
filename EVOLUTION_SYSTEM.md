# EVOLUTION_SYSTEM.md - 自我进化系统

> 项目长效迭代核心引擎  
> 最后更新：2026-05-01

---

## 系统定位

本系统为项目长效迭代核心引擎，基于全流程执行数据、用户反馈、问题记录、效果数据，自动完成规则沉淀、技能优化、系统升级，全程遵循「**无数据不迭代、无验证不升级、无确认不生效**」原则。

---

## 全局进化铁律（不可突破）

1. 所有进化动作**必须基于客观可追溯的数据**，禁止主观脑补、经验迭代、无依据优化
2. 进化规则**必须先验证、再试运行、最终确认后生效**，禁止直接上线未验证规则
3. 进化**仅优化执行效率、修复规则漏洞、补齐能力缺口**，禁止修改顶层体系核心框架
4. 所有进化动作**全程留痕**，进化前后规则对比、验证数据、生效时间，全部归档可查

---

## 四层标准进化流程（固定顺序，不可打乱）

### 第一层：静默全量记录（基础层）

**触发条件**：
- 流程执行全程
- 用户反馈产生
- 问题修复完成
- 审查结果出具时

**执行动作**：
- 自动采集全量客观数据，包括：
  - 技能执行通过率
  - 问题触发频次
  - 反馈类型
  - 修复耗时
  - 效果偏差
  - 流程卡点

**存储规则**：
- 分类归档至 `.workbuddy/knowledge/evolution-data.md`
- 不做任何修改、不干预执行、不触发迭代
- 仅静默沉淀原始数据

---

### 第二层：高频反馈规则固化（进阶层）

**触发条件**：
- 同一类问题、同一类优化建议、同一类卡点，**连续出现≥3次**

**执行动作**：
1. 自动提取共性规律
2. 定位关联技能与节点
3. 生成优化规则补丁
4. 试运行验证无副作用后正式生效

**核心约束**：
- 仅补充细节规则、修复执行漏洞
- **不修改技能核心职责、不改变流程顺序**

**示例**：
```
问题：Agent重复编造API（出现3次）
→ 提取规律：未验证API就写入代码
→ 生成规则：hallucination-prevention.md 添加强制验证步骤
→ 试运行：1个任务验证
→ 生效：更新到skills目录
```

---

### 第三层：低效技能专项优化（优化层）

**触发条件**：
- 单个技能**连续多次执行通过率偏低**
- 问题频发、耗时超标、效果不达标

**执行动作**：
1. 自动分析低效根源
2. 生成专项优化方案
3. 全量验证无冲突后更新技能文档

**核心约束**：
- 仅提升执行质量与效率
- **不改变技能核心定位、不跨职责扩展**

**示例**：
```
技能：code-quality.md
问题：检查清单执行率低（仅60%）
→ 分析：检查项太多，Agent遗漏
→ 优化：精简为4类核心检查点，添加强制输出模板
→ 验证：3个任务测试，执行率提升到95%
→ 生效：更新skills/code-quality.md
```

---

### 第四层：新增能力技能生成（扩展层）

**触发条件**：
- 某一类高频操作、固定流程、重复动作，**反复出现且无对应技能覆盖**

**执行动作**：
1. 自动归纳操作逻辑
2. 生成新技能标准模板
3. 试运行验证闭环后正式纳入执行体系

**核心约束**：
- 新增技能**贴合现有流程、不打破闭环、不重叠现有技能职责**

**示例**：
```
高频操作：问题排查（出现5次，无标准流程）
→ 归纳逻辑：收集证据→分析模式→提出假设→验证修复
→ 生成技能：issue-fixer.md（标准四步法）
→ 验证：2个任务测试，排查时间减少50%
→ 生效：添加到skills目录
```

## 进化系统与子Agent联动规则

1. **反馈观测子Agent**：提供原始反馈数据，为进化系统**唯一数据来源**
2. **进化迭代子Agent**：执行四层进化逻辑、生成优化方案、完成验证测试
3. **主控制Agent**：负责进化规则最终确认、生效管控、版本归档
4. 所有子Agent**必须服从进化后的最新规则**，旧规则同步归档废止

## 进化归档规范

每次进化完成后，自动生成《进化迭代记录》，同步写入 `.workbuddy/knowledge/evolution-data.md`，永久留存可追溯。

### 记录模板

```markdown
**触发原因**：API幻觉问题出现3次

**验证结果**：1个任务测试通过，幻觉率从30%降至0%
**生效时间**：2026-05-01
**归档位置**：.agent/skills/hallucination-prevention.md v1.1
```
