### 问题诊断
经过详细测试和分析，发现以下问题：

1. **价格过滤逻辑问题**：
   - `filterByCategory`函数调用时缺少`headCount`参数，导致计算人均预算错误
   - 修正：将`headCount`参数正确传递给`filterByCategory`函数

2. **极端情况处理**：
   - `calculateMaxPerCapita`函数在人均预算很小时可能导致上限为0
   - 修正：确保人均上限至少为1元

3. **输入验证**：
   - 未检查`headCount`是否大于0，可能导致除零错误
   - 修正：添加`headCount > 0`的检查

4. **资金匹配精度**：
   - 资金来源字符串匹配可能因空格问题失败
   - 修正：使用`.trim()`处理资金来源字符串

### 修复内容

**修改文件**：`src/productListGenerator.ts`

1. **修复`filterByCategory`调用**（第243行）：
   ```typescript
   // 修复前

   // 修复后  
   const candidates = shuffle(filterByCategory(products, category, scene, fundSource, totalBudget, headCount));
   ```

2. **增强`calculateMaxPerCapita`函数**（第53-61行）：
   - 添加字符串`.trim()`处理
   - 确保最小上限为1元
   - 清晰定义折扣系数（工会经费0.8，其他资金0.9）

3. **添加输入验证**（第240-242行）：

   if (headCount <= 0) {

4. **修复`filterByCategory`函数逻辑**（第74-118行）：
