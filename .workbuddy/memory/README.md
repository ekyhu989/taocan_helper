# 角色记忆系统使用指南

> 创建时间：2026-05-01
> 适用对象：所有项目参与者

---

## 📚 记忆架构

### 三层记忆体系

```
.workbuddy/memory/
├── MEMORY.md                      ← 项目级记忆（所有角色共享）
├── pm-memory.md                   ← PM专属记忆
├── designer-memory.md             ← Designer专属记忆
├── developer-memory.md            ← Developer专属记忆
├── frontend-developer-memory.md   ← Frontend-Developer专属记忆
├── qa-memory.md                   ← QA专属记忆
├── devops-memory.md               ← DevOps专属记忆
└── sessions/                      ← 会话记忆归档
    └── [角色]-session-*.md        ← 每次会话记录
```

---

## 🎯 使用场景

### 场景1：新会话启动

**操作步骤：**

1. AI自动读取记忆文件（已在AGENTS.md中配置规则）
2. 或手动提醒AI："请先读取记忆文件"

**AI会自动：**
- 读取 `.workbuddy/memory/MEMORY.md`
- 读取对应角色的记忆文件（如 `pm-memory.md`）
- 读取最新的会话记录（如 `sessions/pm-session-20260501.md`）
- 确认："已加载[角色]记忆和项目记忆，了解当前状态"

---

### 场景2：任务完成

**AI会自动：**

1. 更新角色专属记忆文件
   - 记录任务结果
   - 记录关键决策
   - 记录踩坑经验

2. 更新会话记录
   - 创建/更新 `sessions/[角色]-session-YYYYMMDD.md`
   - 记录本次会话的完整上下文

---

### 场景3：会话快达到上限

**你需要说：**
```
"会话快达到上限了，请更新记忆并保存会话记录"
```

**AI会：**
1. 提炼本次会话的关键经验
2. 更新角色专属记忆文件
3. 保存会话记录到 `sessions/` 目录
4. 提醒："[角色]记忆已保存，可以开启新会话"

---

### 场景4：开启新会话

**新会话第一句话（可选）：**
```
"请读取记忆文件，了解项目当前状态"
```

**AI会自动读取并加载记忆，然后继续工作。**

---

## 📝 记忆文件说明

### MEMORY.md（项目级记忆）

**内容：**
- 项目基本信息
- 核心架构决策
- 全局踩坑记录
- 项目进度状态

**更新频率：** 每周提炼更新，或会话结束时按需更新

**谁读取：** 所有角色

---

### 角色记忆文件（如 pm-memory.md）

**内容：**
- 当前任务状态
- 关键决策记录
- 角色专属踩坑
- 与用户沟通记录
- 待办事项

**更新频率：** 每次任务完成后更新

**谁读取：** 对应角色专属

---

### 会话记录（如 sessions/pm-session-20260501.md）

**内容：**
- 本次会话完成的任务
- 关键决策
- 踩坑记录
- 经验总结
- 遗留问题
- 下次继续的进度

**更新频率：** 每次会话结束前保存

**谁读取：** 对应角色专属

---

## ✅ 最佳实践

### 1. 会话结束前必须更新记忆

**错误做法：**
```
直接关闭会话 ❌
```

**正确做法：**
```
"请更新记忆并保存会话记录" ✅
```

### 2. 新会话开始确认记忆加载

**错误做法：**
```
直接开始新任务 ❌
```

**正确做法：**
```
确认AI已加载记忆后再开始工作 ✅
```

### 3. 定期清理过期记忆

**建议：**
- 每周检查一次角色记忆文件
- 清理已完成的任务
- 提炼关键经验到 MEMORY.md
- 归档旧会话记录

## 🔧 手动操作指南

### 查看记忆文件

```bash
# 查看项目记忆
cat .workbuddy/memory/MEMORY.md

# 查看PM记忆
cat .workbuddy/memory/pm-memory.md

# 查看会话记录
ls .workbuddy/memory/sessions/
cat .workbuddy/memory/sessions/pm-session-20260501.md
```

如果AI没有自动更新，你可以手动提醒：

```
"请更新 pm-memory.md，记录以下内容：
- 任务结果：xxx
- 关键决策：xxx
- 踩坑记录：xxx"
```

## 📊 记忆流转示例

### PM角色完整工作流

```
新会话开始
  ↓
读取 MEMORY.md + pm-memory.md + 最新会话记录
  ↓
确认："已加载PM记忆和项目记忆"
  ↓
执行PM任务（需求分析、任务分解）
  ↓
任务完成 → 更新 pm-memory.md + 会话记录
  ↓
会话快结束
  ↓
用户："请更新记忆并保存会话记录"
  ↓
AI提炼关键经验 → 更新 pm-memory.md
  ↓
保存 sessions/pm-session-20260501.md
  ↓
"PM记忆已保存，可以开启新会话"
```

## 🚨 常见问题

### Q1：记忆文件太多怎么办？

**A：** 定期清理：
- 删除30天前的会话记录
- 提炼角色记忆到 MEMORY.md
- 保持角色记忆文件精简（不超过100行）

### Q2：角色记忆可以共享吗？

**A：** 不可以！角色记忆必须隔离，避免上下文污染。

### Q3：新角色怎么添加记忆文件？

**A：** 复制模板：
```bash
cp .workbuddy/memory/pm-memory.md .workbuddy/memory/[新角色]-memory.md
```
