## 🔄 任务状态流转

```
待开发 → 进行中 → 待审查 → 已完成
```

### 状态定义
- **待开发**: 任务已分配，等待开始
- **进行中**: 开发者正在处理
- **待审查**: 开发完成，等待代码审查
- **已完成**: 审查通过，已合并

## 📝 提交规范

### Git提交信息格式
```
类型: 简要描述

详细说明（可选）
```

### 类型说明
- `feat`: 新功能
- `fix`: Bug修复
- `style`: 样式调整（不影响功能）
- `refactor`: 代码重构
- `docs`: 文档更新

### 示例
```
feat: 实现品单手动调整功能

- 增加商品编辑、删除、添加功能
- 实时计算金额和合规性检查
- 添加"已手动调整"状态标识
```
