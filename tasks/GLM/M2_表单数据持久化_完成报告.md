## 一、任务执行摘要

实现了表单数据自动保存与恢复功能。用户填写的6个基础信息字段和5个公文信息字段，在页面刷新/关闭浏览器后自动恢复，无需重新填写。同时提供"清空表单"按钮一键重置。

## 三、验收标准对照表

### 自动保存

| 验收项 | 状态 | 实现方式 |
|--------|------|----------|
| 基础信息录入页6字段实时保存到 localStorage | ✅ | `useEffect` 监听 `solutionFormData` → `debounce(300ms)` → `saveSolutionForm()` |
| 公文生成页5字段实时保存到 localStorage | ✅ | `useEffect` 监听 `reportFormData` → `debounce(300ms)` → `saveReportForm()` |
| 保存触发时机：防抖 300ms | ✅ | `debounce(fn, 300)` 包装，避免每次击键都写入 |
| localStorage 键名 `taocang_solution_form` | ✅ | `formStorage.ts` 常量 `SOLUTION_FORM_KEY` |
| localStorage 键名 `taocang_report_form` | ✅ | `formStorage.ts` 常量 `REPORT_FORM_KEY` |

### 自动恢复

| 验收项 | 状态 | 实现方式 |
|--------|------|----------|
| 页面加载时自动检查 localStorage | ✅ | `useEffect([], ...)` 挂载时执行 `loadSolutionForm()` + `loadReportForm()` |
| 有保存数据则自动恢复到表单 | ✅ | `setSolutionFormData(saved)` / `setReportFormData(saved)` |
| 恢复时显示"已恢复上次填写的数据" | ✅ | 绿色提示条 + ✓ 图标 |
| 提示 3 秒后自动消失 | ✅ | `setTimeout(() => setShowRestoreNotice(false), 3000)` |

### 清空功能

| 验收项 | 状态 | 实现方式 |
|--------|------|----------|
| 基础信息录入页提供"清空表单"按钮 | ✅ | BasicInfoForm 底部灰色按钮 |
| 公文生成页提供"清空表单"按钮 | ✅ | 公文信息卡片底部灰色按钮 |
| 点击后二次确认 | ✅ | `window.confirm('确定要清空...')` |
| 确认后清空表单并清除 localStorage | ✅ | `setSolutionFormData(defaults)` + `clearSolutionForm()` |
| 清空后显示"表单已清空"提示 | ✅ | 灰色提示条 + 🗑 图标 |

### 场景切换保持

| 验收项 | 状态 | 实现方式 |
|--------|------|----------|
| 录入→预览切换数据不丢失 | ✅ | 数据存储在 React state + localStorage，视图切换不影响 |
| 预览→返回修改数据保持 | ✅ | `handleBackToSolution()` 仅切换 `currentView`，不清空 state |

## 四、技术实现详情

### 4.1 formStorage.ts 模块

```
导出函数:
  saveSolutionForm(data)   → localStorage.setItem('taocang_solution_form', ...)
  loadSolutionForm()       → JSON.parse(localStorage.getItem(...)) 或 null
  saveReportForm(data)     → localStorage.setItem('taocang_report_form', ...)
  loadReportForm()         → JSON.parse(localStorage.getItem(...)) 或 null
  clearSolutionForm()      → localStorage.removeItem('taocang_solution_form')
  clearReportForm()        → localStorage.removeItem('taocang_report_form')
  clearAllForms()          → 同时清除两个键
  debounce(fn, delay=300)  → 防抖工具函数（含 .cancel() 方法）
```

### 4.2 App.jsx 集成点

| 位置 | 内容 |
|------|------|
| import | `useCallback, useRef` + formStorage 全部函数 |
| ref | `debouncedSaveSolution` / `debouncedSaveReport`（防抖实例） |
| useEffect 1 | 初始化防抖实例 + 组件卸载时 cancel |
| useEffect 2 | 监听 `solutionFormData` → 防抖保存 |
| useEffect 3 | 监听 `reportFormData` → 防抖保存 |
| useEffect 4 | 挂载时恢复数据 + 显示恢复提示（3秒自动消失） |
| handleClearSolutionForm | 二次确认 → 重置为默认值 → 清 localStorage → 显示清空提示 |
| handleClearReportForm | 二次确认 → 重置为默认值 → 清 localStorage → 显示清空提示 |
| JSX | 恢复提示（绿色）+ 清空提示（灰色）+ BasicInfoForm onClearForm prop |
| JSX | 公文信息表单底部添加清空按钮 |

## 七、注意事项

1. **防抖安全**：组件卸载时自动 cancel 防抖定时器，防止内存泄漏
2. **错误容忍**：localStorage 读写均 try-catch 包裹，失败只 console.warn 不阻断流程
3. **隐私安全**：仅保存表单输入数据，不保存品单结果或公文内容（历史方案由 M1 管理）
4. **浏览器兼容**：localStorage 在所有现代浏览器中可用，无兼容问题
