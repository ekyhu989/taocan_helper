# AI采购方案生成工具 - 界面开发 - Product Requirement Document

## Overview
- **Summary**: 基于产品需求文档，使用 React + Tailwind CSS 搭建三个核心页面界面：基础信息录入页、采购方案预览页、采购申请报告页。重点关注 UI 还原，使用 Mock 数据，暂不实现复杂业务逻辑。
- **Purpose**: 为 AI 采购方案生成工具提供完整的前端界面框架，满足政务系统的简洁、合规的视觉要求。
- **Target Users**: 政企单位采购经办人员、部门负责人、财务审核人员

## Goals
- 实现基础信息录入表单页面，包含完整的表单字段和合规性校验提示
- 实现采购方案预览页面，包含商品卡片/列表展示和价格计算
- 实现采购申请报告页面，包含合规的公文格式展示
- 实现简单的单页应用视图切换，首页为录入表单，点击"生成方案"切换到预览与报告页
- 实现轻量级表单校验，检查人数和总预算是否为空
- 界面风格符合政务系统规范，简洁专业
- 使用 Mock 数据，暂不连接后端 API

## Non-Goals (Out of Scope)
- 不实现后端 API 连接和数据持久化
- 不实现复杂的表单验证和提交逻辑
- 不实现 AI 服务调用
- 不实现用户认证和权限管理
- 不实现商品库管理功能

## Background & Context
- 已有完整的产品需求文档和开发规范
- 技术栈：React + Ant Design（规范要求），但用户明确要求使用 Tailwind CSS
- 需要严格遵循术语规范，避免使用"购买、下单、支付、商城、购物车"等词汇

## Functional Requirements
- **FR-1**: 基础信息录入页 - 实现包含单位名称、采购场景、人数、总预算、资金来源、申请部门、申请人的完整表单界面
- **FR-2**: 合规性校验提示 - 人均预算超过 500 元时显示黄色小字提醒
- **FR-3**: 采购方案预览页 - 实现商品卡片/列表展示，包含商品名称、价格、数量、小计等信息
- **FR-4**: 采购方案价格计算 - 显示商品合计、人均标准等价格信息
- **FR-5**: 采购申请报告页 - 实现公文格式的纯文本展示，包含标题、正文、消费帮扶提示等内容
- **FR-6**: 单页应用视图切换 - 首页为录入表单，点击"生成方案"按钮切换到预览与报告页
- **FR-7**: 表单轻量级校验 - 点击"生成方案"时检查人数和总预算是否为空，为空时阻止跳转并提示"请填写完整信息"

## Non-Functional Requirements
- **NFR-1**: 界面简洁专业，符合政务系统规范
- **NFR-2**: 响应式设计，支持桌面端访问
- **NFR-3**: 操作流程直观，用户体验良好
- **NFR-4**: 代码结构清晰，便于后续功能扩展

## Constraints
- **Technical**: React + Tailwind CSS，使用 Mock 数据
- **Business**: 严格遵循术语规范，使用"采购、方案、预算、物资"等词汇
- **Dependencies**: 无外部 API 依赖，所有数据本地 Mock

## Assumptions
- 用户已有 React 项目基础架构
- 用户已配置好 Tailwind CSS 环境

## Acceptance Criteria

### AC-1: 基础信息录入页界面完整
- **Given**: 用户访问基础信息录入页
- **When**: 页面加载完成
- **Then**: 显示所有必填表单字段（单位名称、采购场景、人数、总预算、资金来源、申请部门、申请人）
- **Verification**: `human-judgment`
- **Notes**: 检查表单字段是否齐全，布局是否合理

### AC-2: 人均预算超标提示
- **Given**: 用户在基础信息录入页输入人数和总预算
- **When**: 人均预算 > 500 元
- **Then**: 页面显示黄色小字提醒："当前人均标准较高，建议按单位规定核实调整。"
- **Verification**: `human-judgment`
- **Notes**: 检查提示文案和样式是否正确

### AC-3: 采购方案预览页商品展示
- **Given**: 用户访问采购方案预览页
- **When**: 页面加载完成
- **Then**: 显示商品卡片/列表，包含商品名称、价格、数量、小计信息
- **Verification**: `human-judgment`
- **Notes**: 检查商品信息展示是否完整清晰

### AC-4: 采购方案价格汇总
- **Given**: 用户查看采购方案预览页
- **When**: 页面加载完成
- **Then**: 显示商品合计金额、人均标准等汇总信息
- **Verification**: `human-judgment`
- **Notes**: 检查价格计算展示是否正确

### AC-5: 采购申请报告公文格式
- **Given**: 用户访问采购申请报告页
- **When**: 页面加载完成
- **Then**: 显示合规的公文格式，包括标题、正文、消费帮扶提示
- **Verification**: `human-judgment`
- **Notes**: 检查公文格式是否规范，术语使用是否正确

### AC-6: 消费帮扶提示文案
- **Given**: 用户查看采购申请报告页
- **When**: 页面加载完成
- **Then**: 显示标准消费帮扶提示："温馨提示：为完成年度消费帮扶任务，建议在食品类采购中优先选用832平台产品，便于集中完成全年指标。"
- **Verification**: `programmatic`
- **Notes**: 文案必须与规范完全一致

### AC-7: 单页应用视图切换
- **Given**: 用户在基础信息录入页
- **When**: 点击"生成方案"按钮且表单校验通过
- **Then**: 页面切换到预览与报告页，显示采购方案预览和采购申请报告
- **Verification**: `human-judgment`
- **Notes**: 检查页面切换是否流畅

### AC-8: 表单轻量级校验
- **Given**: 用户在基础信息录入页
- **When**: 点击"生成方案"按钮，但人数或总预算为空
- **Then**: 阻止页面跳转，并显示提示"请填写完整信息"
- **Verification**: `human-judgment`
- **Notes**: 检查校验逻辑是否生效
