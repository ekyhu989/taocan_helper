# AI采购方案生成工具 - 界面开发 - Verification Checklist

## 基础信息录入页
- [ ] 表单字段齐全（单位名称、采购场景、人数、总预算、资金来源、申请部门、申请人）
- [ ] 表单布局清晰，字段标签明确
- [ ] 采购场景使用下拉选择组件
- [ ] 数字输入字段（人数、总预算）有适当的输入限制提示
- [ ] 人均预算超过 500 元时显示黄色小字提醒
- [ ] 提醒文案为："当前人均标准较高，建议按单位规定核实调整。"
- [ ] 界面风格简洁专业，符合政务系统规范

## 采购方案预览页
- [ ] 商品卡片/列表展示清晰
- [ ] 每个商品显示：名称、单价、数量、小计
- [ ] 商品数量在 5-8 个之间
- [ ] 价格汇总区域显示商品合计金额
- [ ] 价格汇总区域显示人均标准
- [ ] 价格计算展示正确（Mock 数据预设）
- [ ] 界面风格统一专业

## 采购申请报告页
- [ ] 公文格式规范，包含标题
- [ ] 包含主送单位、正文等公文要素
- [ ] 包含品单列表展示
- [ ] 包含消费帮扶提示
- [ ] 消费帮扶提示文案与规范完全一致："温馨提示：为完成年度消费帮扶任务，建议在食品类采购中优先选用832平台产品，便于集中完成全年指标。"
- [ ] 包含落款等必要元素
- [ ] 术语使用正确，无禁用词汇（购买、下单、支付、商城、购物车）

## 整体界面
- [ ] 三个页面视觉风格统一
- [ ] 配色方案简洁专业（蓝白为主色调）
- [ ] 字体大小层次分明
- [ ] 间距布局合理
- [ ] 桌面端响应式展示良好
- [ ] 所有 Mock 数据正确加载
- [ ] 代码结构清晰，组件划分合理
