## 一、问题描述

### 1.1 现象
- 基础信息录入页：资金来源选择**"专项慰问费"**
- 但下方显示：**"工会经费 8折"**，人均上限¥250.00（200/0.8=250）
- 结果页面三个卡片也都显示：**"工会经费 8折"**

### 1.2 截图分析

**图1 - 基础信息录入页**:
```
资金来源: 专项慰问费 (红框标记)
折扣系数: 工会经费 8折  ← 错误！
人均上限: ¥250.00        ← 错误！
```

**图2 - 方案结果页**:
```
三个卡片均显示:
- 折扣系数: 工会经费 8折  ← 错误！
- 人均上限: ¥250.00       ← 错误！
```

### 1.3 正确逻辑

| 资金来源 | 折扣系数 | 计算方式 |
|---------|---------|---------|
| 工会经费 | 8折 (0.8) | 人均上限 = 人均标准 / 0.8 |
| 专项慰问费 | 9折 (0.9) | 人均上限 = 人均标准 / 0.9 |
| 其他资金 | 9折 (0.9) | 人均上限 = 人均标准 / 0.9 |

**当前数据**:
- 人均标准: ¥200.00
- 资金来源: 专项慰问费
- **实际应显示**: "专项慰问费 9折", 人均上限 = 200/0.9 = ¥222.22
- **错误显示**: "工会经费 8折", 人均上限 = 200/0.8 = ¥250.00

## 二、根因分析

### 2.1 问题定位

**折扣系数判断逻辑错误**：代码中判断资金来源的**条件语句有误**，导致无论选择什么资金来源，都按"工会经费"处理。

### 2.2 可能原因

1. **字符串匹配错误**: 比较时使用了错误的资金来源字符串
2. **条件判断缺失**: 没有正确处理"专项慰问费"分支
3. **默认值错误**: 默认按工会经费处理

### 任务1: 修复基础信息录入页折扣显示

**文件**: `src/components/BasicInfoForm.tsx` 或相关基础信息表单组件

**修复内容**:
```typescript
// 找到折扣系数计算逻辑，修复条件判断

// 错误示例（推测）：
const discountRate = fundingSource === '工会经费' ? 0.8 : 0.8;  // 都是0.8！

// 正确代码：
const getDiscountRate = (fundingSource: string): number => {
  switch (fundingSource) {
    case '工会经费':
      return 0.8;  // 8折
    case '专项慰问费':
    case '其他资金':
    case '行政经费':
      return 0.9;  // 9折
    default:
      return 0.9;  // 默认9折
  }
};

const getDiscountLabel = (fundingSource: string): string => {
  switch (fundingSource) {
    case '工会经费':
      return '工会经费 8折';
    case '专项慰问费':
      return '专项慰问费 9折';
    case '其他资金':
      return '其他资金 9折';
    case '行政经费':
      return '行政经费 9折';
    default:
      return '9折';
  }
};
```
