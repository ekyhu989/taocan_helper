## 一、问题1：预算使用率显示逻辑

### 1.1 问题描述

**当前现象**:
- 人均标准: ¥200.00
- 商品合计: ¥222.00
- 预算使用率: **111.0%** 显示红色警告"超出预算"

**问题分析**:
- 实际支付 = ¥200（用户只需按预算支付，折扣优惠是政策支持）
- 多出的¥22是**福利提升**，不是超预算
- 显示111%会给用户造成恐慌

### 1.2 正确逻辑

```
用户预算: ¥200/人 × 50人 = ¥10,000
商品合计: ¥222/人 × 50人 = ¥11,100
实际支付: ¥10,000（用户按预算支付，享受政策折扣福利）

预算使用率: 100% ✅
福利提升: (222-200)/200 = 11%（额外福利）
```

### 1.3 修复方案

**方案B**：不显示"超出预算"警告，改为"福利提升"

当商品合计 > 人均标准时：
- 不显示红色"超出预算"警告
- 显示绿色"福利提升 XX%"
- 预算使用率显示100%

## 二、修复任务

### 任务1: 修改预算使用率计算逻辑

**文件**: `src/components/SchemeResult.tsx` 或相关汇总组件

**修复内容**:

```typescript
// 当前逻辑（错误）
const usageRate = (totalAmount / totalBudget) * 100; // 111%
const isOverBudget = usageRate > 100;

// 修复后（正确）
const calculateBudgetInfo = (
  totalAmount: number,        // 商品合计
  totalBudget: number,        // 总预算
  perCapitaStandard: number,  // 人均标准
  perCapitaLimit: number      // 人均上限
) => {
  // 实际支付 = 预算金额（用户只需支付预算部分）
  const actualPayment = totalBudget;
  
  // 预算使用率 = 100%（只要不超过上限）
  const isWithinLimit = totalAmount <= (perCapitaLimit * personCount);
  const usageRate = isWithinLimit ? 100 : (totalAmount / totalBudget) * 100;
  
  // 福利提升 = 实际商品价值超出预算的部分
  const welfareBonus = ((totalAmount - totalBudget) / totalBudget) * 100;
  
  return {
    usageRate: 100,  // 始终显示100%，因为用户只支付预算金额
    welfareBonus: Math.max(0, welfareBonus),  // 福利提升百分比
    isOverBudget: totalAmount > (perCapitaLimit * personCount), // 只有超过上限才算超预算
  };
};
```

**UI显示修改**:

```tsx

    </div>
  )}

```
