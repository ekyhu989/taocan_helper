```

```

**核心问题**: 卡片显示的是"人均上限"（313），但用户期望看到"实际人均"（278）

### 任务1：修复卡片"人均总价"显示（P0）

#### 问题
卡片显示 `scheme.totalAmount / personCount` 计算的是"人均上限"，不是实际商品的人均价格。

#### 修复要求

**文件**: `src/components/SchemeCardSlider.tsx` 或相关卡片组件

**当前代码**:
```typescript
<div className="flex justify-between">
  <span className="text-gray-600">人均总价</span>
  <span className="text-xl font-bold text-gray-900">
    ¥{(scheme.totalAmount / scheme.personCount).toFixed(0)}
  </span>
</div>
```

**修复后代码**:
```typescript

  (sum, item) => sum + item.product.price * item.quantity, 
  0

const actualPerCapita = actualTotalAmount / scheme.personCount;

<div className="flex justify-between">
  <span className="text-gray-600">人均总价</span>
  <span className="text-xl font-bold text-gray-900">
    ¥{actualPerCapita.toFixed(2)}
  </span>
</div>
```

### 任务3：修复预算使用率计算（P0）

#### 问题
当前显示1.1%，明显计算错误。

#### 修复要求

**文件**: `src/pages/SchemeResult.tsx` 或相关页面

**当前（错误）**:
```typescript
// 错误：用实际人均 / 总预算
const usageRate = (actualPerCapita / totalBudget) * 100; // 278/25000=1.1%
```

**修复后（正确）**:
```typescript
// 正确：用商品总价 / 总预算
const totalCost = scheme.items.reduce(
  (sum, item) => sum + item.product.price * item.quantity, 
  0
); // 278 * 100人 = 27800
const totalBudget = scheme.totalBudget; // 25000
const usageRate = (totalCost / totalBudget) * 100; // 27800/25000=111.2%

// 显示逻辑
<div>
  <span>预算使用率: {usageRate.toFixed(1)}%</span>
  {usageRate > 100 && (
    <span className="text-red-500 ml-2">⚠️ 超出预算</span>
  )}
</div>
```

**验收标准**:
- [ ] 预算使用率计算正确（27800/25000=111.2%）
- [ ] 超出预算时显示警告
