## 🔄 协作流程

### 1. 任务分配流程

```
项目经理创建任务文档
    ↓
分配给对应Agent
    ↓
Agent确认并执行
    ↓
完成后提交任务日志
    ↓
项目经理验收
    ↓
通过/打回修改
```

### 2. 代码提交规范

```bash
# 提交信息格式
type: subject

# type类型
feat:     新功能
fix:      修复
docs:     文档
style:    格式
refactor: 重构
test:     测试
chore:    构建/工具

# 示例
git commit -m "feat: V2.0-001 实现价格约束算法"
git commit -m "fix: V2.0-002 修复移动端按钮错位"
```

### 3. 代码审查要点

| 检查项 | GLM | Doubao | DeepSeek |

| 测试覆盖 | ✅ | - | ✅ |

| 响应式适配 | - | ✅ | - |

## 📊 沟通规范

### 1. 日报格式

```
【Agent名称】V2.0 开发日报 - YYYY-MM-DD

今日完成:
1. {任务编号} - {任务名称} - {进度}%
2. ...

明日计划:
1. {任务编号} - {计划}
2. ...

阻塞问题:
- {问题描述} - 需要{谁}协助
```

### 2. 问题升级机制

| 级别 | 响应时间 | 处理方式 |
|------|---------|---------|
| P0-阻塞 | 2小时内 | 立即讨论解决 |
| P1-重要 | 当天 | 日报中说明 |
| P2-一般 | 次日 | 任务备注记录 |
