## 部署步骤

### 第一步：环境准备
1. **确认腾讯云账号状态**
   - 登录腾讯云控制台
   - 确认云开发（CloudBase）服务已开通
   - 确认静态网站托管服务已开启

2. **本地环境检查**
   ```bash
   # 检查Node.js版本
   node --version  # 需要 >= 16.0.0
   
   # 检查npm版本
   npm --version   # 需要 >= 8.0.0
   
   # 检查CloudBase CLI
   cloudbase --version  # 需要 >= 3.0.0
   ```

### 第二步：构建生产版本
```bash
# 1. 安装依赖（如果尚未安装）
npm install

# 2. 构建生产版本
npm run build

# 3. 验证构建结果
ls -la dist/
# 应包含：index.html, assets/目录
```

### 第三步：部署到腾讯云
```bash
# 1. 登录腾讯云CloudBase
cloudbase login

# 2. 初始化部署配置（如果首次部署）
cloudbase init

# 3. 部署到静态网站托管
cloudbase hosting:deploy dist/ -e <环境ID>
```

**环境ID说明**：
- 测试环境：test-env-xxxx
- 生产环境：prod-env-xxxx

### 第四步：域名配置（可选）
1. **自定义域名**
   - 在CloudBase控制台添加自定义域名
   - 配置DNS解析（CNAME记录）
   - 等待SSL证书自动签发（约5-10分钟）

2. **CDN加速配置**
   - 开启全球CDN加速
   - 配置缓存策略（建议：静态资源缓存30天）
   - 开启Gzip压缩

### 第五步：功能验证
部署完成后，访问网站进行功能验证：

| 功能模块 | 验证步骤 | 预期结果 |
|---------|---------|---------|
| 政策文库 | 1. 访问政策库页面<br>2. 点击政策链接<br>3. 测试搜索功能 | 1. 页面正常加载<br>2. 跳转官方源地址<br>3. 搜索结果正确 |
| 公文生成 | 1. 填写表单生成公文<br>2. 测试Word导出<br>3. 测试PDF导出 | 1. 公文生成成功<br>2. Word文件正常下载<br>3. PDF文件正常下载 |
| 合规测算 | 1. 输入预算数据<br>2. 切换832核算口径<br>3. 测试预警提示 | 1. 计算正确<br>2. 口径切换正常<br>3. 预警颜色正确 |
| 数据安全 | 1. 访问设置页面<br>2. 测试数据备份<br>3. 测试数据恢复 | 1. 页面正常加载<br>2. 备份成功提示<br>3. 恢复流程完整 |

## 回滚方案

### 回滚条件
- 生产环境出现严重功能缺陷（P0级）
- 性能问题导致服务不可用
- 安全漏洞被暴露

### 回滚步骤
1. **立即回滚到V1.5**
   ```bash
   # 切换到V1.5标签
   git checkout v1.5
   
   # 重新构建并部署
   npm run build
   cloudbase hosting:deploy dist/ -e prod-env-xxxx
   ```

2. **数据恢复**
   - 从CloudBase恢复最新备份数据
   - 验证数据完整性
   - 通知用户可能的短暂数据丢失

3. **问题排查**
   - 分析V1.6问题原因
   - 修复缺陷
   - 重新测试后择机再次发布
