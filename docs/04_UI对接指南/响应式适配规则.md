# 响应式适配规则 - V1.5

> 设计师：Kimi2.5

---

## 📐 断点定义

采用 **Mobile-First** 设计策略，定义以下断点：

| 断点名称 | 宽度范围 | 设备类型 | 布局策略 |
|---------|---------|---------|---------|
| **xs** | < 640px | 手机竖屏 | 单列布局，紧凑间距 |
| **sm** | 640px - 767px | 手机横屏/大手机 | 双列网格，适中间距 |
| **md** | 768px - 1023px | 平板竖屏 | 双列网格，宽松间距 |
| **lg** | 1024px - 1279px | 平板横屏/小桌面 | 左右分栏，功能卡片2×2 |
| **xl** | 1280px - 1439px | 桌面显示器 | 左右分栏，特性展示 |
| **2xl** | ≥ 1440px | 大桌面/宽屏 | 最大宽度限制，居中布局 |

```css
/* Tailwind CSS 断点 */
module.exports = {
  theme: {
    screens: {
      'sm': '640px',
      'md': '768px',
      'lg': '1024px',
      'xl': '1280px',
      '2xl': '1440px',
    },
  },
}
```

## 📱 各断点详细规范

### 1. XS 断点 (< 640px) - 手机竖屏

**目标设备**：iPhone SE/12/13/14/15 竖屏，Android 手机

**布局结构**：
```
┌─────────────────────────────────┐
│           Header                │  56px
├─────────────────────────────────┤
│                                 │
│         Hero 区域               │
│      单列，居中布局              │
│                                 │
├─────────────────────────────────┤
│      核心功能（2×2网格）         │
│    ┌──────┐    ┌──────┐        │
│    │ 卡片1 │    │ 卡片2 │        │
│    └──────┘    └──────┘        │
│    ┌──────┐    ┌──────┐        │
│    │ 卡片3 │    │ 卡片4 │        │
│    └──────┘    └──────┘        │
├─────────────────────────────────┤
│         CTA按钮                 │
│      全宽，底部固定              │
├─────────────────────────────────┤
│         Footer                  │
├─────────────────────────────────┤
│         底部导航栏              │  64px
└─────────────────────────────────┘
```

**尺寸规范**：

| 元素 | 尺寸/间距 | 说明 |
|-----|----------|------|
| 页面边距 | 16px (px-4) | 左右内边距 |
| Hero区域内边距 | 32px 0 24px | 上下内边距 |
| 主标题 | 24px | font-size: 24px |
| 副标题 | 16px | font-size: 16px |
| 描述文字 | 14px | font-size: 14px |
| 功能卡片网格 | 2列，间距12px | grid-cols-2, gap-3 |
| 卡片高度 | 120px | 固定高度 |
| 卡片圆角 | 12px | rounded-xl |
| CTA按钮高度 | 48px | h-12 |
| CTA按钮圆角 | 24px | rounded-full |
| 底部导航高度 | 64px | 含安全区 |

**代码示例**：
```jsx

  <h1 className="text-2xl font-bold text-center">智能采购助手</h1>

    {features.map(f => (
      <div key={f.id} className="h-[120px] rounded-xl p-4">
        {/* 卡片内容 */}
      </div>
    ))}
  </div>
  
  <button className="w-full h-12 rounded-full mt-6">
    立即制定方案
  </button>
</div>
```
