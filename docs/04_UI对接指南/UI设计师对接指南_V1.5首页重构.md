> 本文档为UI设计师（Kimi2.5）提供项目背景、约束条件和设计需求，确保在保持现有功能完整的基础上进行UI/UX升级。

## 📋 项目背景

### 产品名称
**智能采购助手** - 新疆工会采购合规方案生成工具

### 产品定位
- 为新疆地区机关单位工会提供采购方案制定、公文生成、合规测算的一站式工具
- 完全本地化处理，保护用户隐私
- 符合政府采购政策要求，支持832平台采购

### 目标用户
- 新疆地区机关单位工会工作人员
- 财务部门采购经办人员
- 单位采购审批人员

---

## 🎯 V1.5升级目标

### 核心目标
将现有的"直接进入功能"模式，升级为"首页导航+功能模块"模式，提升：
1. **首次使用体验** - 用户知道产品能做什么
2. **功能发现性** - 快速找到需要的功能
3. **操作便捷性** - 随时在各模块间切换

### 升级范围
```
新增：
├── 首页（Landing Page）- 全新设计
│   ├── 项目简介区域
│   ├── 4大功能入口卡片
│   └── 快速开始按钮
│
└── 底部导航栏（全局）
    ├── 首页
    ├── 制定方案
    ├── 合规测算
    └── 更多

优化：
├── 页面切换过渡动画
├── 返回首页快捷方式
└── 响应式布局细节
```

## ⚠️ 设计约束条件（红线）

### 1. 功能完整性约束
| 约束项 | 要求 | 原因 |
|--------|------|------|
| **现有页面不动** | Page1-4的布局、字段、校验逻辑保持不变 | 已通过测试，不能回退 |
| **数据流不变** | Page1→Page2→Page3的数据传递逻辑不变 | 核心业务逻辑 |
| **合规性不变** | 所有合规提示、政策引用保持 | 政策要求 |

### 2. 技术约束
| 约束项 | 要求 |
|--------|------|
| **技术栈** | React + Tailwind CSS + Vite |
| **样式变量** | 使用现有CSS变量，不重新定义 |
| **组件规范** | 遵循现有组件命名和结构 |

### 3. 设计约束
| 约束项 | 要求 |
|--------|------|
| **配色** | 主色调蓝色 #1890ff，保持品牌一致性 |
| **字体** | 思源黑体（标题Bold/正文Regular） |
| **图标** | 使用Emoji或开源图标库，保持简洁 |
| **移动端优先** | 设计以移动端为主，适配桌面端 |

## 🎨 设计需求详解

### 1. 首页（Landing Page）

#### 页面结构
```
┌─────────────────────────────────┐
│  顶部：Logo + 产品名称           │
├─────────────────────────────────┤
│  简介区：                        │
│  "智能采购助手"                  │
│  "新疆工会采购合规方案生成工具"   │
│  一句话描述产品价值               │
├─────────────────────────────────┤
│  功能入口区（4个卡片网格）:       │
│  ┌─────────┐  ┌─────────┐       │
│  │ 📝      │  │ 📊      │       │
│  │制定方案 │  │合规测算 │       │
│  │简要描述 │  │简要描述 │       │
│  └─────────┘  └─────────┘       │
│  ┌─────────┐  ┌─────────┐       │
│  │ 📚      │  │ 📦      │       │
│  │政策文库 │  │商品库   │       │
│  │简要描述 │  │简要描述 │       │
│  └─────────┘  └─────────┘       │
├─────────────────────────────────┤
│  快速开始：大按钮"立即制定方案"  │
├─────────────────────────────────┤
│  底部：版本号 + 帮助链接         │
└─────────────────────────────────┘
```

#### 功能卡片设计
每个卡片包含：
- **图标**：Emoji或图标（📝📊📚📦）
- **标题**：功能名称（制定方案/合规测算/政策文库/商品库）
- **描述**：一句话说明（如"快速生成合规采购方案"）
- **点击效果**：hover状态、点击反馈

#### 交互逻辑
- 点击卡片 → 跳转到对应功能页面
- "立即制定方案"按钮 → 直接进入Page1
- 首次进入显示首页，后续可记住用户偏好

---

### 2. 底部导航栏（全局组件）

#### 导航项
```
┌──────┬──────┬──────┬──────┐
│  🏠  │  📝  │  📊  │  ➕  │
│ 首页 │ 方案 │ 合规 │ 更多 │
└──────┴──────┴──────┴──────┘
```

#### 交互逻辑
- **首页**：返回Landing Page
- **方案**：直接进入Page1（制定方案）
- **合规**：进入Page4（合规测算）
- **更多**：展开菜单（政策文库/商品库/历史方案/设置）

#### 显示规则
- 在首页显示完整导航
- 在功能页面（Page1-4）显示简化导航（保留返回首页）
- 当前页面对应导航项高亮

---

### 3. 页面切换动画

#### 过渡效果
- **首页→功能页**：从右向左滑入
- **功能页→首页**：从左向右滑出
- **功能页之间**：淡入淡出

#### 动画参数
- 时长：300ms
- 缓动：ease-in-out
- 保持流畅，不卡顿
