## 🔐 HTTPS证书配置

### 证书类型
- **主证书**: Let's Encrypt免费证书（自动续期）
- **备用证书**: 腾讯云SSL证书（商业证书）
- **通配符证书**: *.taocang.com（支持所有子域名）

### 证书配置步骤

#### 1. 腾讯云CloudBase证书配置

```bash
# 登录腾讯云控制台
cloudbase login

# 配置域名证书
cloudbase hosting domain add taocang.com --cert-type auto
cloudbase hosting domain add www.taocang.com --cert-type auto
cloudbase hosting domain add staging.taocang.com --cert-type auto
```

#### 2. Vercel证书配置

```bash
# 配置Vercel域名
vercel domains add taocang.com
vercel domains add staging.taocang.com

# 验证域名所有权
vercel domains verify taocang.com
```

#### 3. 证书自动续期配置

创建证书自动续期脚本：

```bash
#!/bin/bash
# scripts/renew-certificates.sh

# 检查证书有效期
cert_expiry=$(openssl x509 -in /path/to/cert.pem -noout -enddate | cut -d= -f2)
current_date=$(date +%s)
expiry_date=$(date -d "$cert_expiry" +%s)
days_until_expiry=$(( (expiry_date - current_date) / 86400 ))

if [ $days_until_expiry -lt 30 ]; then
    echo "证书即将过期，开始续期..."
    
    # 腾讯云证书续期
    cloudbase hosting domain renew taocang.com
    
    # Vercel证书续期
    vercel domains renew taocang.com
    
    echo "证书续期完成"
else
    echo "证书有效期充足，无需续期"
fi
```

## 🛡️ 安全配置

### HTTP安全头配置

在Vite配置中添加安全头：

```javascript
// vite.config.js
export default defineConfig({
  // ...其他配置
  server: {
    headers: {
      'Strict-Transport-Security': 'max-age=31536000; includeSubDomains',
      'X-Frame-Options': 'DENY',
      'X-Content-Type-Options': 'nosniff',
      'Referrer-Policy': 'strict-origin-when-cross-origin',
      'Content-Security-Policy': "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'"
    }
  }
});
```

## 📊 监控配置

### 域名健康监控

创建域名健康检查脚本：

```javascript
// scripts/domain-health-check.js
const domains = [
  'https://taocang.com',
  'https://www.taocang.com',
  'https://staging.taocang.com',
  'https://dev.taocang.com'
];

async function checkDomain(domain) {
  try {
    const response = await fetch(domain, { 
      method: 'HEAD',
      timeout: 10000 
    });
    
    return {
      domain,
      status: response.status,
      https: domain.startsWith('https'),
      timestamp: new Date().toISOString()
    };
  } catch (error) {
    return {
      domain,
      status: 'ERROR',
      error: error.message,
      timestamp: new Date().toISOString()
    };
  }
}

// 定期检查
export async function monitorDomains() {
  const results = await Promise.all(domains.map(checkDomain));
  console.log('域名健康检查结果:', results);
  
  // 发送告警
  const errors = results.filter(r => r.status === 'ERROR' || r.status >= 400);
  if (errors.length > 0) {
    // 发送告警通知
    sendAlert(errors);
  }
}

// 每5分钟检查一次
setInterval(monitorDomains, 5 * 60 * 1000);
```

## 🔧 故障排除

### 常见问题及解决方案

#### 1. 证书验证失败
**问题**: 证书验证失败，域名无法访问
**解决方案**:
```bash
# 检查DNS解析
dig taocang.com

# 重新验证域名
cloudbase hosting domain verify taocang.com
vercel domains verify taocang.com
```

#### 2. HTTPS重定向循环
**问题**: 网站陷入重定向循环
**解决方案**:
- 检查服务器配置中的重定向规则
- 清除浏览器缓存和Cookie
- 验证证书配置是否正确

#### 3. 混合内容警告
**问题**: 页面包含HTTP资源，导致安全警告
**解决方案**:
- 检查所有资源链接是否为HTTPS
- 更新第三方库的CDN链接
- 使用相对路径或协议相对URL

### 紧急恢复流程

#### 证书失效紧急处理
1. 立即启用备用证书
2. 联系证书提供商紧急续期
3. 更新DNS记录到备用服务器
4. 通知用户临时访问备用域名

#### 域名被劫持处理
1. 立即暂停域名解析
2. 联系域名注册商锁定域名
3. 检查DNS记录是否被篡改
4. 启用备用域名提供服务
