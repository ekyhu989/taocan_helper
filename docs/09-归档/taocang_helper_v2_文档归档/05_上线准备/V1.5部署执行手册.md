# V1.5 部署执行手册

> 本文档指导如何将智能采购助手V1.5版本部署到腾讯云静态网站托管。

---

## 📋 部署前检查清单

### 1. 代码状态确认

- [ ] DeepSeek的V1.5-4和V1.5-5任务已完成并验收通过
- [ ] 所有代码已提交到Git
- [ ] 本地构建测试通过

### 2. 环境准备

- [ ] 腾讯云账号已注册并完成实名认证
- [ ] 本地已安装Node.js 16+
- [ ] 本地已安装腾讯云CLI（可选，用于命令行部署）

---

## 🚀 部署步骤

### 步骤1: 本地构建

```bash
# 1. 进入项目目录
cd e:/taocang_helper

# 2. 确保依赖最新
npm install

# 3. 构建生产版本
npm run build

# 4. 验证构建结果
ls dist/
# 应该看到: index.html, assets/, images/, products.json
```

**构建成功标志**:
- 无红色错误信息
- `dist` 目录生成
- `dist/index.html` 存在

---

### 步骤2: 腾讯云环境准备

#### 方式A: 云开发（推荐）

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 进入 **云开发 CloudBase**
3. 点击 **新建环境**
   - 环境名称: `taocang-helper`
   - 计费方式: 按量付费（有免费额度）
4. 等待环境创建完成（约1-2分钟）
5. 进入环境 → 静态网站托管 → 开启静态网站托管

#### 方式B: COS对象存储

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 进入 **对象存储 COS**
3. 点击 **创建存储桶**
   - 名称: `taocang-helper`
   - 地域: 选择"北京"（离新疆较近）
   - 访问权限: 公有读私有写
4. 进入存储桶 → 基础配置 → 静态网站 → 开启

---

### 步骤3: 上传文件

#### 方式A: 控制台上传（简单）

1. 进入云开发/COS控制台
2. 找到静态网站托管的文件管理
3. 点击 **上传文件** 或 **上传文件夹**
4. 选择 `e:/taocang_helper/dist` 目录下的所有文件
5. 等待上传完成

#### 方式B: 命令行部署（推荐）

```bash
# 1. 安装腾讯云CLI
npm install -g @cloudbase/cli

# 2. 登录腾讯云
tcb login

# 3. 部署到云开发
tcb hosting:deploy dist -e taocang-helper

# 或使用COS CLI
coscmd config -a <SecretId> -s <SecretKey> -b taocang-helper -r ap-beijing
coscmd upload -r dist/ /
```

---

### 步骤4: 配置访问

1. 在控制台获取 **默认访问链接**
   - 格式: `https://taocang-helper-xxx.tcloudbaseapp.com`
   
2. 浏览器测试访问
   - 确认首页正常显示
   - 确认4个功能入口可点击
   - 确认底部导航栏正常

3. 微信测试
   - 将链接发送到微信
   - 在微信内打开测试
   - 确认加载速度（应1-2秒）

---

### 步骤5: （可选）配置自定义域名

如需使用自己的域名：

1. 购买域名（腾讯云/阿里云/万网）
2. 域名备案（国内服务器必须备案，约7-20天）
3. 腾讯云控制台 → 添加自定义域名
4. 配置DNS解析
5. 申请免费SSL证书
6. 等待生效

## ⚠️ 常见问题

### Q1: 上传失败
**解决**: 检查文件大小，分批上传；或使用命令行部署

### Q2: 访问404
**解决**: 确认 `index.html` 在根目录；检查静态网站配置

### Q3: 微信打不开
**解决**: 检查域名是否被微信拦截；使用腾讯云默认域名通常没问题

### Q4: 更新不生效
**解决**: 清除浏览器缓存；等待CDN刷新（5-10分钟）
