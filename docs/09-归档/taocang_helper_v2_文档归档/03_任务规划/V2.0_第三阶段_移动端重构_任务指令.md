### 任务2：振动反馈机制

**任务编号**: Doubao-V2.0-005
**优先级**: P0
**预计工时**: 2天
**截止日期**: 2026-04-19

#### 需求描述
实现设备振动反馈机制，支持嘈杂现场环境下的盲操判断。

#### 具体要求

1. **振动模式定义**:
   ```typescript
   enum VibrationPattern {
     SUCCESS = 'success',      // 短振1次 - 操作成功
     ERROR = 'error',          // 长振2次 - 合规拦截/错误
     WARNING = 'warning',      // 间歇短振 - 二次确认
     PROGRESS = 'progress'     // 轻微振动 - 操作进行中
   }

   const vibrationPatterns = {
     success: [50],                    // 50ms
     error: [200, 100, 200],          // 200ms振, 100ms停, 200ms振
     warning: [30, 50, 30, 50, 30],   // 间歇短振
     progress: [20]                   // 轻微振动
   };
   ```

2. **触发场景**:
   | 场景 | 振动模式 | 说明 |
   |------|---------|------|
   | 保存成功 | success | 方案保存完成 |
   | 生成成功 | success | 方案生成完成 |
   | 合规拦截 | error | 红色预警触发 |
   | 价格超限 | error | 超出价格约束 |
   | 二次确认 | warning | 删除/覆盖确认 |
   | 重新生成 | warning | 数据清空风险 |
   | 数据加载 | progress | 异步加载中 |

3. **用户控制**:
   - 设置中可开启/关闭振动
   - 默认开启
   - 记住用户偏好

4. **兼容性处理**:
   - 检测设备是否支持振动
   - 不支持时降级为提示音（可选）
   - iOS/Android差异处理

#### 验收标准
- [ ] 4种振动模式实现
- [ ] 触发场景完整覆盖
- [ ] 用户可控制开关
- [ ] 兼容性处理完善
- [ ] 构建通过，无错误

#### 文件变更
| 文件 | 变更类型 | 说明 |
|------|---------|------|
| `src/utils/vibration.ts` | 新增 | 振动工具函数 |
| `src/hooks/useVibration.ts` | 新增 | 振动Hook |

### 任务3：现场应急模式

**任务编号**: Doubao-V2.0-006
**优先级**: P1
**预计工时**: 3天
**截止日期**: 2026-04-21

#### 需求描述
实现10秒快速创建草稿功能，支持离线缓存和跨设备续编。

#### 具体要求

1. **快速创建流程**:
   ```
   步骤1: 点击"应急模式"按钮
   步骤2: 填写总预算（数字键盘）
   步骤3: 填写人数（数字键盘）
   步骤4: 选择场景（4选1）
   步骤5: 自动生成极简方案
   总耗时: ≤ 10秒
   ```

2. **极简方案内容**:
   - 自动匹配3-5个商品
   - 总价控制在预算范围内
   - 标记为"应急草稿"
   - 包含基础合规检查

3. **草稿管理**:
   - 自动保存到历史列表
   - 标记"未完成"状态
   - 支持随时续编
   - 支持转为正式方案

4. **离线支持**:
   - 无网络可正常创建
   - 本地缓存商品库
   - 有网络时自动同步

5. **跨设备续编**:
   - 导出草稿文件
   - 在其他设备导入
   - 数据完整保留

#### 验收标准
- [ ] 10秒内完成创建
- [ ] 自动生成极简方案
- [ ] 草稿状态正确标记
- [ ] 离线功能正常
- [ ] 跨设备续编可用
- [ ] 构建通过，无错误
