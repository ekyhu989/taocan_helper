## 二、详细需求

### 2.1 30秒全局撤销

**功能描述**：
- 批量操作后显示撤销按钮，30秒内可一键撤销
- 撤销后恢复操作前状态

**支持的操作**：
- 批量删除商品
- 批量修改价格
- 批量删除历史方案
- 清空回收站

**UI设计**：
```
┌─────────────────────────────────────────┐
│ ✅ 已删除5个商品                         │
│ [撤销]              29秒后消失          │
└─────────────────────────────────────────┘
```

**实现逻辑**：
```typescript
interface UndoableAction {
  id: string;
  type: 'delete' | 'update' | 'clear';
  timestamp: number;
  beforeData: any;
  afterData: any;
  undo: () => void;
}

const undoStack: UndoableAction[] = [];

const executeWithUndo = (action: UndoableAction) => {
  // 执行操作
  action.execute();
  // 加入撤销栈
  undoStack.push(action);
  // 30秒后移除
  setTimeout(() => {
    removeFromUndoStack(action.id);
  }, 30000);
};
```

### 2.2 全量操作日志

**记录内容**：
```typescript
interface OperationLog {
  id: string;
  timestamp: string;
  operation: string;
  target: string;
  beforeValue: any;
  afterValue: any;
  amountChange?: number;
  userAgent: string;
}
```

**记录场景**：
- 商品增删改
- 历史方案复用/修改/删除
- 批量操作
- 合规预警触发

**日志导出**：
- 支持导出JSON/Excel
- 包含完整操作记录
- 用于审计追溯

### 2.3 前端数据二次校验

**校验规则**：
| 数据类型 | 校验规则 |
|---------|---------|
| 单价 | > 0 且 < 10000 |
| 数量 | > 0 且 < 10000 |
| 预算 | > 0 且 < 1000000 |
| 人数 | > 0 且 < 10000 |

**拦截方式**：
- 输入时实时校验
- 提交前完整校验
- 异常值红色提示

- 恢复默认商品库
