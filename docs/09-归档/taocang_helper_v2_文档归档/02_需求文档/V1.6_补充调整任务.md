## 任务2：方案生成逻辑优化

### 问题描述
当前逻辑：预算200元，生成的套餐人均金额达到400多元，严重超标。
根本原因：生成算法没有根据资金来源设置合理的价格上限。

### 调整方案

**新增约束逻辑**：根据资金来源设置套餐人均金额上限

| 资金来源 | 折扣系数 | 计算公式 | 示例（预算200元） |
|---------|---------|---------|------------------|
| 工会经费 | 8折 | 上限 = 预算 ÷ 0.8 | 200 ÷ 0.8 = **250元** |
| 其他资金 | 9折 | 上限 = 预算 ÷ 0.9 | 200 ÷ 0.9 ≈ **222元** |

**实现要求**：
1. 在商品筛选时增加价格上限约束
2. 套餐组合时确保人均金额不超过上限
3. 如果无法生成符合预算的套餐，给出友好提示

### 验收标准
- [ ] 工会经费200元预算，生成套餐人均≤250元
- [ ] 其他资金200元预算，生成套餐人均≤222元
- [ ] 无法生成时给出明确提示（如"当前预算下无法生成满足条件的套餐，建议调整预算或手动选择商品"）
- [ ] 不同资金来源正确应用对应折扣系数

### 涉及文件
```
修改：src/utils/schemeGenerator.ts     # 核心生成逻辑
修改：src/components/SchemeGenerator.jsx # 提示信息展示
```

## 任务3：合规测算功能重构

### 问题描述
用户反馈合规测算功能"鸡肋"，对实际工作帮助不大，使用率/价值感低。

### 待决策方案

需要用户确认最终方案：

| 方案 | 功能定位 | 实现复杂度 | 用户价值 |
|------|---------|-----------|---------|
| A-预算规划助手 | 帮助用户合理规划采购预算，智能推荐资金分配 | 中 | 高 |
| B-政策速查 | 快速检索适用政策条款，关键词搜索 | 低 | 中 |
| C-合规检查清单 | 采购前自检必填项，确保材料完整 | 低 | 中 |
| D-直接移除 | 将空间留给更有价值的功能 | 低 | - |

**建议方案**：A-预算规划助手（与用户核心需求"方案生成"关联最强）

### 如选择方案A-预算规划助手

**功能描述**：
1. 输入：总人数、总预算、资金来源、活动类型
2. 输出：预算分配建议（人均金额、分类占比、合规风险提示）
3. 智能推荐：根据历史数据推荐合理的预算区间

**验收标准**
- [ ] 输入参数后生成预算规划建议
- [ ] 显示人均金额、分类占比（食品/日用品/慰问品）
- [ ] 自动进行合规校验（500元预警、2000元拦截等）
- [ ] 提供优化建议（如"建议增加832平台商品占比至30%"）

修改：src/App.jsx                              # 路由调整
