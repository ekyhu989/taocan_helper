## 🔄 自动化部署

### 1. GitHub Actions配置

创建自动化部署工作流：

```yaml
# .github/workflows/deploy.yml
name: V2.0 Deploy to CloudBase

on:
  push:
    branches: [main]
  
jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
    
    - name: Install dependencies
      run: npm ci
    
    - name: Run tests
      run: npm test
    
    - name: Build project
      run: npm run build
    
    - name: Deploy to CloudBase
      run: |
        npm install -g @cloudbase/cli
        cloudbase hosting deploy dist -e ${{ secrets.CLOUDBASE_ENV_ID }}
      env:
        CLOUDBASE_ENV_ID: taocang-helper-6g9ad4x31a5351d5
```

### 2. 本地自动化脚本

使用已创建的部署脚本：

## 🐛 故障排除

### 常见问题及解决方案

#### 1. 构建失败
**问题**: `npm run build` 失败
**解决方案**:
```bash
# 清理缓存重新安装
rm -rf node_modules package-lock.json
npm cache clean --force
npm ci

# 检查TypeScript错误
npx tsc --noEmit
```

#### 2. 部署失败
**问题**: CloudBase部署报错
**解决方案**:
```bash
# 检查登录状态
cloudbase login --check

# 检查环境权限
cloudbase env list

# 重新登录
cloudbase login
```

#### 3. 应用访问异常
**问题**: 部署后应用无法访问
**解决方案**:
- 检查构建产物完整性
- 验证环境变量配置
- 检查控制台错误信息
