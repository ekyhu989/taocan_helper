3.1 任务1：基础信息录入页
优先级： 高
3.1.1 功能点
•单位名称输入框
•采购场景下拉选择（节日福利/职工慰问/简易福利礼品）
•人数输入框（数字验证）
•总预算输入框（数字验证）
•资金来源输入框（仅作为文本标签）
•申请部门输入框
•申请人输入框
3.1.2 交互逻辑
•输入验证：必填项校验
•实时计算：总预算 ÷ 人数 = 人均预算
•合规提醒：人均＞500元时显示黄色提示
3.2 任务2：智能品单生成
优先级： 高
3.2.1 功能点
•根据场景推荐商品组合
•显示商品名称、单价、数量、小计
•支持增删改商品
•自动计算合计金额
•显示832平台商品标识
3.2.2 算法逻辑
•基于商品库数据，按场景分类推荐
•随机组合4-6个商品
•按比例分配金额
•不做复杂的"凑预算"算法
3.3 任务3：公文生成与导出
优先级： 高
3.3.1 功能点
•选择公文模板（根据采购场景）
•自动填充变量内容
•生成完整的采购申请报告
•支持导出Word/PDF格式
3.3.2 模板管理
•预设三套模板（见第4部分）
•支持模板更新维护
3.4 任务4：商品库管理
优先级： 中
3.4.1 功能点
•商品信息维护（名称、价格、单位、分类、推荐标识）
•支持832平台商品标识
•本地JSON数据存储
•商品分类管理
