## 功能清单

### F1: 历史方案管理增强（P1）

#### 需求背景
V1.5已支持历史方案保存，但随着使用增多，用户需要快速查找特定方案。当前列表仅按时间倒序排列，缺乏搜索和筛选能力。

#### 功能需求

**1.1 搜索功能**
- 搜索框放置在历史方案列表顶部
- 支持按单位名称搜索（模糊匹配）
- 支持按采购场景搜索
- 实时搜索（输入时即时过滤）
- 空搜索结果显示友好提示

**1.2 筛选功能**
- 筛选条件：
  - 采购场景（全部/传统节日慰问/专项活动物资/精准帮扶慰问）
  - 日期范围（最近7天/30天/90天/全部）
  - 预算范围（全部/500元以下/500-1000元/1000元以上）
- 筛选结果实时更新
- 显示当前筛选条件和结果数量
- 一键清除筛选

**1.3 批量操作**
- 支持多选历史方案（checkbox）
- 批量删除功能（需二次确认）
- 全选/取消全选
- 显示已选择数量

**1.4 收藏功能**
- 单个方案收藏/取消收藏
- 收藏方案置顶显示
- 收藏状态持久化到localStorage

#### 技术实现建议
```javascript
// 搜索过滤逻辑
const filteredHistory = historyItems.filter(item => {
  const matchSearch = item.unitName.includes(searchKeyword) || 
                      item.scene.includes(searchKeyword);
  const matchScene = filterScene === 'all' || item.scene === filterScene;
  const matchDate = isWithinDateRange(item.createdAt, filterDateRange);
  return matchSearch && matchScene && matchDate;
});
```

#### 验收标准
- [ ] 搜索功能实时响应，延迟<300ms
- [ ] 筛选条件组合使用正常
- [ ] 批量删除有确认弹窗
- [ ] 收藏状态持久化
- [ ] 空搜索结果有友好提示

### F2: 数据导出功能（P1）

#### 需求背景
工会工作人员需要定期上报采购数据，或备份重要方案。当前仅支持单个方案的Word/PDF导出，缺乏批量导出和Excel格式支持。

#### 功能需求

**2.1 历史方案导出Excel**
- 导出按钮放置在历史方案页面
- 支持导出单条方案为Excel
- 支持批量导出多条方案为Excel
- Excel包含字段：
  - 基本信息：单位名称、采购场景、人数、总预算、日期
  - 商品清单：商品名称、单价、数量、金额
  - 汇总信息：合计金额、人均标准、832占比

**2.2 采购清单独立导出**
- 在方案预览页面添加"导出清单"按钮
- 单独导出商品清单为Excel/PDF
- 清单格式简洁，适合打印和张贴

**2.3 统计报表导出**
- 在合规测算页面添加"导出报表"按钮
- 导出年度采购统计报表
- 包含：年度预算、已完成金额、完成率、建议采购金额

#### Excel格式规范
```
Sheet1: 基本信息
- 单位名称 | 采购场景 | 人数 | 总预算 | 日期

Sheet2: 商品清单
- 序号 | 商品名称 | 单价 | 数量 | 单位 | 金额 | 是否832平台

Sheet3: 汇总信息
- 合计金额 | 人均标准 | 832平台金额 | 832占比
```

#### 技术实现建议
使用 `xlsx` 库（已安装）生成Excel文件：
```javascript
import * as XLSX from 'xlsx';

const exportToExcel = (data, filename) => {
  const ws = XLSX.utils.json_to_sheet(data);
  const wb = XLSX.utils.book_new();
  XLSX.utils.book_append_sheet(wb, ws, 'Sheet1');
  XLSX.writeFile(wb, `${filename}.xlsx`);
};
```

#### 验收标准
- [ ] Excel文件正常生成和下载
- [ ] 中文内容无乱码
- [ ] 数字格式正确（金额保留2位小数）
- [ ] 批量导出文件名包含日期
- [ ] 大文件导出有loading提示

### F3: 性能优化（P1）

#### 需求背景
当前打包文件 `index-DnHUq9LJ.js` 大小为1.59MB，首屏加载较慢，需要进行代码分割和优化。

#### 优化目标
- 首屏加载时间 < 3秒（4G网络）
- 打包体积减少 30%+
- Lighthouse性能评分 > 80

#### 优化措施

**3.1 代码分割（Code Splitting）**
- 按路由分割代码（每个页面独立chunk）
- 组件懒加载（React.lazy + Suspense）
- 第三方库按需加载

**3.2 资源优化**
- 启用Gzip压缩
- 图片资源优化（如有）
- CSS提取和压缩

**3.3 加载优化**
- 添加骨架屏（Skeleton Screen）
- 首屏关键CSS内联
- 预加载关键资源

#### 技术实现建议
```javascript
// 路由懒加载
const HomePage = React.lazy(() => import('./pages/HomePage'));
const CompliancePage = React.lazy(() => import('./pages/CompliancePage'));

// vite.config.js 配置
export default {
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom'],
          utils: ['./src/utils/historyStorage', './src/utils/formStorage']
        }
      }
    }
  }
}
```

#### 验收标准
- [ ] 打包后JS文件大小 < 1MB
- [ ] 首屏加载时间 < 3秒
- [ ] 路由切换无卡顿
- [ ] Lighthouse性能评分 > 80

### F4: 用户体验优化（P2）

#### 4.1 加载状态
- 按钮点击后显示loading状态
- 页面切换显示过渡动画
- 数据加载显示骨架屏

#### 4.2 空状态处理
- 历史方案为空时显示引导提示
- 商品库为空时显示添加引导
- 搜索结果为空时显示友好提示

#### 4.3 Toast通知
- 操作成功提示（保存、导出、删除）
- 操作失败提示（错误信息）
- 自动消失，不干扰用户

#### 验收标准
- [ ] 所有异步操作有loading状态
- [ ] 空状态有友好提示和引导
- [ ] Toast通知样式统一
