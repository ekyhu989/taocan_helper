### 任务6：批量操作安全防护体系（P0 - 已完成）

**验收标准**：
- ✅ 30秒撤销：批量操作后30秒内可撤销
- ✅ 操作日志：所有高危操作记录完整
- ✅ 前端校验：极端数值无法提交
- ✅ 高危确认：删除等操作需文字确认

**创建的文件**：
- `src/utils/undoManager.ts` - 30秒撤销管理器
- `src/components/UndoToast.jsx` - 撤销提示组件
- `src/components/DangerConfirmModal.jsx` - 高危操作确认弹窗
- `src/utils/dataValidator.ts` - 数据校验工具

**功能说明**：
1. **30秒全局撤销**：
   - 支持批量删除、批量修改价格、清空回收站等操作撤销
   - 操作后显示撤销按钮，30秒倒计时
   - 撤销后恢复操作前状态

2. **全量操作日志**：
   - 记录所有操作（商品增删改、历史方案操作、批量操作等）
   - 支持导出 JSON 和 CSV 格式
   - 包含操作时间、操作类型、变更前后数据

3. **前端数据二次校验**：
   - 单价：0.01 - 10000元
   - 数量：1 - 10000份
   - 预算：0.01 - 1000000元
   - 人数：1 - 10000人

4. **高危操作弹窗确认**：
   - 删除商品、清空回收站、删除历史方案等操作需确认
   - 支持文字确认（输入"确认删除"）
   - 显示操作影响范围

## 四、技术要点

### 任务1 - 导航优化
1. **响应式设计**：
   - 移动端（<1024px）：固定5项导航
   - 桌面端（≥1024px）：6项导航（含历史方案）
2. **合规预警**：
   - 检测人均>2000、人均>500、832占比<30%
   - 有风险时显示红色闪烁按钮
   - 点击跳转诊断页
3. **诊断页**：
   - 风险概览卡片
   - 风险详情列表
   - 核算逻辑展开/收起
   - 一键整改按钮

### 任务3 - 历史方案升级
1. **多维筛选**：6个维度支持组合筛选，实时过滤历史方案列表
2. **标签管理**：预设标签 + 自定义标签（最多5个），标签保存到localStorage
3. **数据对比**：高亮差异，自动生成调整建议，确认后复用方案
4. **操作日志**：记录所有操作（查看/复用/删除），包含时间戳和详细信息

### 任务4 - 商品库升级
1. **回收站**：软删除机制，保留7天，支持恢复和永久删除
2. **批量防护**：改价>100%强制二次确认，防止误操作
3. **节日模板**：春节/古尔邦节/肉孜节预置商品，导入时自动去重
4. **832置顶**：832标识商品在列表和搜索结果中优先显示
5. **排序优化**：832商品置顶，非832商品按更新时间倒序

### 任务6 - 批量操作安全防护体系
1. **撤销机制**：
   - 撤销栈管理，最多保留30秒
   - 支持批量操作的完整撤销
   - 自动清理过期撤销记录
2. **操作日志**：
   - 最多保留1000条记录
   - 支持JSON/CSV导出
   - 记录用户Agent信息
3. **数据校验**：
   - 实时输入校验
   - 提交前完整校验
   - 清晰的错误提示
4. **高危确认**：
   - 支持文字确认
   - 显示操作影响范围
   - 二次确认机制

### 任务7 - 移动端体验优化
1. **大字模式**：
   - CSS类名管理，支持动态切换
   - 局部样式覆盖，不影响整体布局
   - 设置持久化到localStorage
2. **加载状态**：
   - 轻量级CSS动画实现
   - 支持自定义提示信息
   - 三种尺寸适配不同场景
3. **响应式优化**：
