## 四、技术实现详情

### 4.1 historyStorage.ts - 统计函数扩展

```typescript
// 新增接口
interface YearlyProgressData {
  total: number;       // 累计采购金额（元）
  rate: number;        // 完成率（百分比）
  remaining: number;   // 距离30%目标还差多少元
}

// 获取指定年份的累计金额
export function getYearlyTotal(year?: number): number {
  const history = loadHistory();
  return history
    .filter(item => new Date(item.createdAt).getFullYear() === year)
    .reduce((sum, item) => sum + (item.summary?.totalAmount || 0), 0);
}

// 获取年度进度数据
export function getYearlyProgress(yearlyBudget: number): YearlyProgressData {
  const total = getYearlyTotal();
  if (yearlyBudget <= 0) return { total, rate: 0, remaining: 0 };
  
  const rate = (total / yearlyBudget) * 100;
  const targetAmount = yearlyBudget * 0.3; // 30%达标线
  const remaining = Math.max(0, targetAmount - total);
  
  return { total, rate, remaining };
}
```

```

├─ useEffect([yearlyBudget]): 调用 getYearlyProgress() 更新状态

```

## 九、业务规则说明

### 9.1 30%达标线的政策依据

根据《基层工会经费收支管理办法》及新疆工会消费帮扶政策要求：

- 年度福利支出中，**脱贫地区农副产品（832平台）采购金额占比应≥30%**
- 此功能帮助采购人员实时掌握本年度消费帮扶任务完成进度
- 未达标时给出明确的金额建议，辅助决策

### 9.2 计算公式

```
年度完成率 (%) = (本年度已采购总额 ÷ 年度预算总额) × 100

达标差额 (元) = max(0, 年度预算总额 × 30% - 本年度已采购总额)
```

**示例计算**：
```
年度预算：¥100,000
本年度已采购：¥25,000（来自3个历史方案）

完成率 = 25,000 ÷ 100,000 × 100 = 25.0%
达标差额 = max(0, 100,000 × 0.3 - 25,000) = ¥5,000

→ 显示红色警告："还需采购 ¥5,000 元以达标30%"
```

## 十、注意事项

1. **数据实时性**：
   - 每次组件渲染都从localStorage读取最新数据
   - 不缓存历史记录（确保删除/新增后即时反映）

2. **边界情况处理**：
   - 无历史记录：组件不渲染（return null）
   - yearlyBudget ≤ 0：显示默认值但不报错
   - localStorage损坏：由loadHistory()的try-catch处理，返回空数组

3. **性能考量**：
   - useEffect依赖数组`[yearlyBudget]`避免不必要的重复计算
   - 数据量级：最多50条历史记录（M1限制），计算开销可忽略

4. **用户体验**：
