## 背景说明

### 策略调整原因
原商品库允许用户直接添加自定义商品，但存在以下问题：
1. 用户可能绕过主推商品，影响商品推销效果
2. 自定义商品质量无法保证，可能存在合规风险
3. 缺少与用户的进一步沟通机会

### 新策略目标
- **主推商品优先**: 引导用户使用预设商品库
- **创造沟通机会**: 用户需添加商品时联系客服
- **保证商品质量**: 所有商品由官方审核后入库

## 任务内容

### 1. 商品库管理页调整

**当前问题**: 页面有"添加商品"功能，用户可直接添加

**调整后**:
- 移除"添加商品"按钮和相关表单
- 添加客服引导区域：
  ```
  💡 需要添加新商品？
  请联系客服，我们将为您导入到商品库
  [ 📞 联系客服 ]
  ```
- "联系客服"按钮点击后显示联系方式弹窗

**涉及文件**:
```
修改：src/components/ProductLibrary.jsx
```

---

### 2. 方案生成页调整

**当前问题**: 直接读取商品库生成方案，无引导

**调整后**:
- 优先匹配商品库商品生成方案
- 当商品库无法满足需求时，显示引导：
  ```
  ⚠️ 当前商品库暂无完全匹配您需求的商品
  
  建议：
  1. 调整预算或场景条件，重新生成
  2. 联系客服添加您需要的商品到商品库
  
  [重新生成] [联系客服]
  ```

**涉及文件**:
```
修改：src/components/SchemeGenerator.jsx
修改：src/productListGenerator.ts（调整匹配逻辑）
```

---

### 3. 客服联系方式配置

**需求**: 在多处页面提供统一的客服联系方式

**实现方式**:
- 创建客服配置常量：`src/config/serviceConfig.ts`
- 配置内容：
  ```typescript
  export const SERVICE_CONFIG = {
    phone: '400-xxx-xxxx',  // 客服电话
    wechat: 'xxxxxx',       // 客服微信号
    email: 'service@example.com',
    workHours: '工作日 9:00-18:00'
  };
  ```
- 创建联系客服弹窗组件：`src/components/ContactServiceModal.jsx`

**涉及文件**:
```
新增：src/config/serviceConfig.ts
新增：src/components/ContactServiceModal.jsx
```

### 4. 数据同步Bug修复

**问题描述**: 方案生成页和商品库管理页数据不同步

**排查要点**:
1. 检查 `productStorage.ts` 的加载逻辑
2. 确认 `loadProducts()` 是否在每次使用时重新读取
3. 检查 localStorage 写入和读取是否一致
4. 验证页面切换时数据是否正确刷新

**涉及文件**:
```
修改：src/utils/productStorage.ts（如有问题）
修改：src/components/SchemeGenerator.jsx（确保重新加载）
修改：src/components/ProductLibrary.jsx（确保重新加载）
```
