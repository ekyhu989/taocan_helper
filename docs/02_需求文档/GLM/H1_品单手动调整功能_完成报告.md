## 一、任务执行摘要

在现有的方案预览页面（SolutionPreview）中成功增加了商品手动调整功能。用户现在可以：
- **编辑** 商品数量（内联输入框，实时更新小计）
- **删除** 商品（二次确认防误删）
- **添加** 商品（展开式选择面板，排除已选商品）
- **恢复** 系统自动生成的原始方案（一键重置）

所有金额（总合计、832占比、人均福利价值）在每次操作后**立即重新计算**，合规性检查**实时生效**。

## 四、技术实现详情

### 4.1 新增辅助函数 `recalculateSolution`

**文件**: `src/productListGenerator.ts`

```typescript
export function recalculateSolution(
  items: ProductListItem[],
  totalBudget: number
): Pick<ProductListResult, 'totalAmount' | 'budgetUsage' | 'perCapitaAmount' | 'platform832Amount' | 'platform832Rate'>
```

- 接收修改后的 `items` 数组和原始总预算
- 重新计算：总金额、预算使用率、人均金额、832金额及占比
- 纯函数，无副作用

### 4.2 App.jsx 新增状态

```javascript
const [isAdjusted, setIsAdjusted] = useState(false);           // 是否手动调整
const [originalProductListResult, setOriginalProductListResult] = useState(null); // 原始方案快照
```

### 4.3 App.jsx 新增回调函数（5个）

| 函数 | 触发时机 | 核心逻辑 |
|------|----------|----------|
| `handleAdjustProduct(id, qty)` | 编辑确认 | 更新单个商品数量 + 重算汇总 |
| `handleRemoveProduct(id)` | 删除确认 | 过滤商品 + 重算汇总（不允许删空） |
| `handleAddProduct(product)` | 添加商品 | 追加新商品（默认数量1）+ 重算汇总 |
| `handleResetSolution()` | 恢复按钮 | 从快照恢复 + 清除调整标记 |
| `handleUpdateProductItems(items)` | 通用更新 | 统一入口，所有调整操作都经过这里 |

3. **商品排序**: 添加新商品追加到列表末尾，暂不支持拖拽排序。
