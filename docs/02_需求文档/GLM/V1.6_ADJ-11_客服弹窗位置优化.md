## 问题描述

### 问题1：客服弹窗位置
客服联系弹窗（ContactServiceModal）当前位置太靠页面底部，用户需要向下滚动才能看到，容易被忽略。

**现状**：弹窗显示在页面内容区域底部  
**期望**：弹窗居中显示在视口中央，无需滚动即可看到

### 问题2：移动端视口配置
页面在手机上可以任意缩放，用户容易误操作缩小页面，导致内容变小、难以阅读。

**现状**：`minimum-scale` 未设置，允许缩小  
**期望**：禁止缩小（minimum-scale=1.0），允许放大（maximum-scale=2.0）

---

## 调整方案

### 修改前（当前样式）

弹窗可能使用了相对定位或默认流式布局，导致跟随页面内容滚动。

### 修改后（期望样式）

弹窗应该：
1. **固定在视口中央**（fixed + center）
2. **背景遮罩覆盖全屏**（防止误操作）
3. **点击遮罩可关闭**（提升用户体验）

### 具体实现

**修改文件**: `src/components/ContactServiceModal.jsx`

**关键样式调整**:

```jsx
// 遮罩层 - 全屏固定
<div className="fixed inset-0 bg-black bg-opacity-50 z-50 flex items-center justify-center"
     onClick={onClose}>
  
  {/* 弹窗内容 - 居中显示 */}
  <div className="bg-white rounded-lg shadow-xl max-w-md w-full mx-4 p-6"
       onClick={(e) => e.stopPropagation()}>
    
    {/* 弹窗头部 */}
    <div className="flex items-center justify-between mb-4">
      <h3 className="text-lg font-bold text-gray-900">联系客服</h3>
      <button onClick={onClose} className="text-gray-400 hover:text-gray-600">
        <svg className="w-6 h-6" fill="none" stroke="currentColor" viewBox="0 0 24 24">
          <path strokeLinecap="round" strokeLinejoin="round" strokeWidth={2} d="M6 18L18 6M6 6l12 12" />
        </svg>
      </button>
    </div>
    
    {/* 弹窗内容 */}
    <div className="space-y-4">
      {/* 客服信息 */}
    </div>
    
  </div>
</div>
```

### 关键CSS类说明

| 类名 | 作用 |
|------|------|
| `fixed inset-0` | 固定定位，覆盖整个视口 |
| `bg-black bg-opacity-50` | 半透明黑色遮罩 |
| `z-50` | 确保在最上层 |
| `flex items-center justify-center` | Flex布局，水平和垂直居中 |
| `max-w-md w-full mx-4` | 最大宽度限制，响应式边距 |
| `onClick={(e) => e.stopPropagation()}` | 阻止冒泡，防止点击弹窗内容时关闭 |

## 第二部分：移动端视口配置优化

### 调整方案

**修改前（当前配置）**:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

**问题**：
- 允许任意缩放（包括缩小）
- 用户容易误触缩小页面
- 页面内容可能变小难以阅读

**修改后（优化配置）**:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, minimum-scale=1.0, maximum-scale=2.0, user-scalable=yes" />
```

**优化效果**：
| 特性 | 设置 | 效果 |
|------|------|------|
| 初始缩放 | 1.0 | 正常显示 |
| 最小缩放 | 1.0 | **禁止缩小** |
| 最大缩放 | 2.0 | 允许放大到2倍 |
| 用户缩放 | yes | 允许手动缩放 |

### 实现说明

- **禁止缩小**：页面始终填满屏幕，不会变小
- **允许放大**：用户可以放大查看商品详情、金额等
- **防止误操作**：避免双指捏合导致页面缩小
- **符合移动端最佳实践**：大多数现代Web应用采用此方案
