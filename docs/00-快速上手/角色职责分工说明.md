## 🎯 前后端分工

### ❌ 常见混淆

**错误理解：**
- "developer包含前端和后端"
- "前端开发也交给developer"

**正确理解：**
- `developer` = **后端**开发工程师
- `frontend-developer` = **前端**开发工程师
- 两个角色**完全独立**

---

### ✅ 明确分工

#### 前端开发工程师（frontend-developer）

**负责：**
- ✅ 微信小程序页面开发（wxml/wxss/js）
- ✅ 管理后台前端开发（Vue/React）
- ✅ UI实现（HTML/CSS/JavaScript）
- ✅ 交互逻辑（事件处理、状态管理）
- ✅ 响应式适配（PC/平板/手机）
- ✅ 组件开发与复用

**不负责：**
- ❌ 云函数开发
- ❌ 数据库操作
- ❌ API设计
- ❌ 后端业务逻辑

**可访问目录：**
```
✅ src/miniapp/        # 小程序代码
✅ src/admin/          # 管理后台前端
✅ docs/               # 文档
```

**禁止访问目录：**
```
❌ src/cloudfunctions/ # 云函数（后端）
❌ database/           # 数据库
❌ src/backend/        # 后端代码
```

---

#### 后端开发工程师（developer）

**负责：**
- ✅ 云函数开发（Node.js/Python）
- ✅ 数据库操作（增删改查）
- ✅ API设计与实现
- ✅ 后端业务逻辑
- ✅ 数据处理与转换
- ✅ 第三方服务集成

**不负责：**
- ❌ 小程序页面开发
- ❌ 管理后台前端开发
- ❌ UI实现
- ❌ CSS样式编写

**可访问目录：**
```
✅ src/cloudfunctions/ # 云函数
✅ database/           # 数据库脚本
✅ src/backend/        # 后端代码
✅ docs/               # 文档
```

**禁止访问目录：**
```
❌ src/miniapp/        # 小程序代码（前端）
❌ src/admin/src/      # 管理后台前端（前端）
```

## 🔧 协作流程

### 前端与后端的协作

```
Frontend-Developer（前端）          Developer（后端）
        ↓                                ↓
   需要API接口                    设计API接口
        ↓                                ↓
   提出API需求 ────────────→  实现API接口
        ↓                                ↓
   调用API接口  ←────────────  提供API文档
        ↓                                ↓
   前端数据处理                    后端数据处理
        ↓                                ↓
   UI展示                        数据库存储
```

### PM协调前后端

```
PM任务指令：

【前端任务】
- 负责人: Frontend-Developer
- 任务: 实现报告上传页面
- 依赖: 需要API接口（T006）
- 输入: UI设计稿、API文档

【后端任务】
- 负责人: Developer
- 任务: 实现AI分析云函数
- 依赖: 无
- 输入: 需求文档、数据模型

PM确保：
1. 后端先完成API接口
2. 前端再调用API
3. 前后端联调测试
```

## ⚠️ 常见错误

### 错误1：把前端任务分配给developer

**错误：**
```
任务：实现小程序页面
负责人: Developer  ❌ 错误！
```

**正确：**
```
任务：实现小程序页面
负责人: Frontend-Developer  ✅ 正确！
```

---

### 错误2：让frontend-developer修改数据库

**错误：**
```
任务：修改数据库表结构
负责人: Frontend-Developer  ❌ 错误！
```

**正确：**
```
任务：修改数据库表结构
负责人: Developer  ✅ 正确！
```

---

### 错误3：前后端职责混淆

**错误：**
```
developer负责所有开发工作（前后端）
```

**正确：**
```
frontend-developer负责前端
developer负责后端
两者协作完成功能
```

## 📋 PM任务分配指南

### 前端任务特征

如果任务包含以下内容，分配给 **Frontend-Developer**：
- ✅ 页面开发
- ✅ UI实现
- ✅ 样式编写（CSS）
- ✅ 交互逻辑（JavaScript）
- ✅ 组件开发
- ✅ 响应式适配
- ✅ 用户体验优化

### 后端任务特征

如果任务包含以下内容，分配给 **Developer**：
- ✅ 云函数开发
- ✅ 数据库操作
- ✅ API设计
- ✅ 业务逻辑
- ✅ 数据处理
- ✅ 算法实现
- ✅ 第三方服务集成

## 🎯 总结

### 核心原则

1. **前后端分离** - 职责清晰，各司其职
2. **专业化** - 前端专注UI/UX，后端专注逻辑/数据
3. **协作** - 通过API接口对接
4. **PM协调** - PM负责任务分解和分配
