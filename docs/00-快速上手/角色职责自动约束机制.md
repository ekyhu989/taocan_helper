# 🤖 角色职责自动约束机制

> 框架自动强制角色遵守职责边界，无需用户手动提醒

---

## 🎯 设计原则

**核心理念：** 自动化 ≠ 用户提醒，而是框架内置强制约束

### ❌ 错误的方式（用户负担）
```
用户："请以PM身份，只完成需求阶段，不要继续其他阶段"
```

### ✅ 正确的方式（框架自动约束）
```
框架内置规则 → AI自动遵守 → 用户零负担
```

---

## 🔒 自动约束机制

### 1. 角色职责边界（硬约束）

每个角色配置文件中都有**不可违反的规则**：

#### PM角色（product-manager.json）
```json
"critical_rules": [
  "规则13：【自动停止】PM角色只负责需求阶段，完成后必须停止并等待用户审核",
  "规则14：【禁止越界】PM禁止创建设计文档、技术方案、执行计划",
  "规则15：【强制输出】完成后必须输出：'✅ 需求阶段完成，请审核。审核通过后，请说：进入设计阶段'"
]
```

#### Designer角色（designer.json）
```json
"critical_rules": [
  "规则7：【自动停止】Designer角色只负责设计阶段，完成后必须停止并等待用户审核",
  "规则8：【禁止越界】Designer禁止创建技术方案、代码文件",
  "规则9：【强制输出】完成后必须输出：'✅ 设计阶段完成，请审核。审核通过后，请说：进入方案阶段'"
]
```

#### Developer角色（developer.json）
```json
"critical_rules": [
  "规则12：【自动停止】每个阶段完成后必须停止并等待用户审核",
  "规则13：【禁止越界】Developer禁止跳过审核直接进入下一阶段",
  "规则14：【强制输出】每个阶段完成后必须输出：'✅ [阶段名]完成，请审核'"
]
```

---

### 2. 权限控制（物理隔离）

```json
"permissions": {
  "allowed_directories": ["docs/01-需求文档/"],
  "forbidden_directories": ["docs/02-设计文档/", "docs/03-技术方案/", "src/"],
  "allowed_operations": ["read", "write"],
  "forbidden_operations": ["deploy", "execute_code"]
}
```

**效果：**
- PM只能写入 `docs/01-需求文档/`
- 尝试写入其他目录会被框架拒绝

---

### 3. 阶段门禁（HARD-GATE）

```
需求阶段 → [HARD-GATE] → 等待用户审核 → 审核通过 → 设计阶段
                                                    ↓
                                            [HARD-GATE]
                                                    ↓
                                            等待用户审核
                                                    ↓
                                              审核通过 → 方案阶段
```

**规则11（PM）：** 规格未批准前，禁止进入实现阶段（HARD-GATE）

---

### 4. 检查清单（强制流程）

```json
"checklists": {
  "after_write": [
    "【HARD-GATE】保存规格文档并请求用户审查",
    "【HARD-GATE】等待用户批准后才进入实现阶段",
    "【自动停止】输出完成提示后，必须停止，禁止继续执行其他阶段",
    "【禁止越界】禁止创建设计文档、技术方案、代码文件"
  ]
}
```

## 📋 完整工作流（自动化）

### 阶段1：需求阶段（PM）

**用户输入：**
```
我想做速检云V3.0，需求如下：
1. 新增AI检测报告分析
2. 优化检测流程
```

**PM自动执行：**
1. ✅ 读取归档报告
2. ✅ 生成需求规格说明书
3. ✅ 回答office-hours 6个问题
4. ✅ 完成CEO review 10维度审查
5. ✅ **自动停止** ⬅️ 框架强制

**PM输出：**
```
✅ 需求阶段完成，已生成：
📄 docs/01-需求文档/V3.0_需求规格说明书.md

请审核。审核通过后，请说：进入设计阶段
```

**然后PM停止，等待用户审核。**

---

### 阶段2：设计阶段（Designer）

**用户输入：**
```
进入设计阶段
```

**Designer自动执行：**
1. ✅ 读取需求规格说明书
2. ✅ 生成设计规范文档
3. ✅ **自动停止** ⬅️ 框架强制

**Designer输出：**
```
✅ 设计阶段完成，已生成：
📄 docs/02-设计文档/V3.0_设计规范文档.md

请审核。审核通过后，请说：进入方案阶段
```

**然后Designer停止，等待用户审核。**

---

### 阶段3：方案阶段（Developer）

**用户输入：**
```
进入方案阶段
```

**Developer自动执行：**
1. ✅ 读取需求文档 + 设计文档
2. ✅ 生成技术架构方案
3. ✅ **自动停止** ⬅️ 框架强制

**Developer输出：**
```
✅ 方案阶段完成，已生成：
📄 docs/03-技术方案/V3.0_技术架构方案.md

请审核。审核通过后，请说：进入计划阶段
```

**然后Developer停止，等待用户审核。**

## 🔍 如果AI越界了怎么办？

### 场景：AI一次性完成了多个阶段

**原因：**
- AI忽略了角色配置文件中的约束规则
- 或者当前环境没有正确加载角色配置

**解决方案：**

1. **检查角色配置是否加载：**
   ```
   请确认你已加载product-manager.json角色配置
   ```

2. **提醒AI遵守规则：**
   ```
   请遵守规则13：完成后必须停止，禁止继续执行其他阶段
   ```

3. **删除越界创建的文档：**
   ```
   删除docs/02-设计文档/和docs/03-技术方案/
   你越界了，请只完成需求阶段
   ```

4. **重新启动：**
   ```
   请以PM角色重新开始，只生成需求规格说明书
   ```

## ✅ 验证自动化是否生效

### 测试方法

1. **启动PM角色：**
   ```
   请以PM角色身份，帮我生成V3.0需求规格说明书
   ```

2. **观察行为：**
   - ✅ 正确：生成需求文档后停止，输出"请审核"
   - ❌ 错误：继续创建设计文档、技术方案

3. **如果错误：**
   - 检查角色配置文件是否正确加载
   - 确认`critical_rules`是否生效
   - 可能需要调整配置或提示词

## 🎯 总结

| 特性 | 实现方式 | 效果 |
|------|---------|------|
| 角色边界 | `critical_rules` | 硬约束，不可违反 |
| 自动停止 | `after_write`检查清单 | 完成后强制停止 |
| 阶段门禁 | HARD-GATE规则 | 未审核不能进入下一阶段 |
| 权限控制 | `permissions`配置 | 物理隔离目录访问 |
| 强制输出 | 标准化提示语 | 明确告知用户下一步 |

**核心优势：**
- ✅ 用户零负担（不需要手动提醒）
- ✅ 框架自动约束（内置规则）
- ✅ 物理隔离（权限控制）
- ✅ 质量保障（阶段审核）

**这就是真正的自动化！** 🚀
