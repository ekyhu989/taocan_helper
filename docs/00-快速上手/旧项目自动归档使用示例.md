> 一键归档旧项目，直接与PM沟通v2.0需求

---

## 🎯 使用场景

你有一个旧项目（例如：速检云v1.0），想要：
1. 保留所有旧代码作为参考
2. 自动分类归档到框架中
3. 生成项目现状报告
4. **直接与PM开始沟通v2.0需求**

## 🔧 高级用法

### 1. 忽略特定目录

编辑 `archive_legacy_project.py`，修改 `ignore_patterns`：

```python
ignore_patterns = shutil.ignore_patterns(
    'node_modules',
    '.git',
    '__pycache__',
    '*.pyc',
    '.DS_Store',
    'dist',          # 构建产物
    'build',
    '.next',
    '.nuxt',
    'logs',          # 日志文件
    'temp',          # 临时文件
)
```

### 2. 只归档部分文件

如果旧项目太大，可以只归档关键文件：

```bash
# 手动创建归档目录
mkdir -p docs/09-归档/su-jian-yun-v1-key-files

# 只复制关键文件
cp -r E:\old-projects\su-jian-yun-v1\src docs/09-归档/su-jian-yun-v1-key-files/
cp E:\old-projects\su-jian-yun-v1\README.md docs/09-归档/su-jian-yun-v1-key-files/
cp E:\old-projects\su-jian-yun-v1\package.json docs/09-归档/su-jian-yun-v1-key-files/
```

### 3. 多个旧项目归档

```bash
# 归档项目A
python archive_legacy_project.py /path/to/project-a

# 归档项目B
python archive_legacy_project.py /path/to/project-b

# 两个项目都会归档到docs/09-归档/
```

## ⚠️ 注意事项

### 1. 归档不会影响原始项目

归档是**复制**操作，原始项目完全保留。

### 2. 自动排除大目录

脚本会自动排除：
- `node_modules/` - 依赖包
- `.git/` - Git历史
- `dist/` - 构建产物
- `__pycache__/` - Python缓存

### 3. 归档后验证

归档完成后，建议：
1. 检查归档目录是否完整
2. 查看现状报告是否准确
3. 确认可以访问关键文件
