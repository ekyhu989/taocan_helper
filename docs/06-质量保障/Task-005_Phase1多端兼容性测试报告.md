### 5.2 P3 Low级别问题

#### BUG-001: CSS @import 顺序不规范

| 字段 | 内容 |
|------|------|
| **严重级别** | P3 - Low |
| **影响范围** | 旧版Safari（<14）、部分国产浏览器 |
| **复现步骤** | 1. 构建项目 `npm run build`  
               2. 观察构建警告: `@import must precede all other statements` |
| **问题描述** | `src/index.css` 中 `@import` 语句位于 `@tailwind` 指令之后，违反CSS规范。当前因Vite处理正常，但在严格CSS解析环境中可能导致tokens.css和global.css未能正确加载 |
| **建议修复** | 将 `@import` 移到文件最顶部（`@tailwind` 之前），或通过JS/TS在main.tsx中直接import |
| **影响平台** | P3(Safari) |

#### BUG-002: 构建产生空chunk

| 字段 | 内容 |
|------|------|
| **严重级别** | P3 - Low |
| **影响范围** | 生产构建产物 |
| **复现步骤** | 1. 执行 `npm run build`  
               2. 观察输出: `Generated an empty chunk: "pdf"`, `"docx"`, `"crypto"` |
| **问题描述** | `vite.config.ts` 中 `manualChunks` 配置了 `pdf`、`docx`、`crypto` 独立chunk，但当前代码中未使用这些依赖，导致产生0字节空chunk |
| **建议修复** | 移除未使用的manualChunks配置项，或在代码中实际使用后再配置 |
| **影响平台** | 所有平台 |
