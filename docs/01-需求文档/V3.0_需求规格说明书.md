## 🎯 一、需求概述

### 1.1 项目背景

淘仓助手（AI采购方案生成工具）V2.0已上线运行，主要服务基层工会采购场景。经过审查发现以下核心问题：

1. **部署问题**：Vercel部署后微信内置浏览器无法访问
2. **设计问题**：AI化设计严重，不符合政企场景专业要求
3. **功能问题**：公文模板单一、政策文件检索体验差
4. **体验问题**：交互反馈不足、操作路径长、缺少引导

### 1.2 V3.0目标

**核心定位**：从"AI玩具"升级为"政企级专业工具"

**关键指标**：
- 微信访问成功率：0% → 100%
- 用户完成率（进入到导出）：35% → 70%
- 平均使用时长：15分钟 → 8分钟
- 用户满意度：5/10 → 8/10

### 1.3 需求范围

#### Must Have（必须实现）- P0
- [M1] 修复Vercel微信访问问题
- [M2] 全新设计系统（去AI化）
- [M3] 公文模板库与自定义选择
- [M4] 政策文件分类检索系统
- [M5] 交互体验优化（加载/反馈/引导）

#### Should Have（应该实现）- P1
- [S1] 性能优化（首屏加载<2s）
- [S2] 表单体验优化（实时验证/自动保存）
- [S3] 移动端手势操作
- [S4] Toast提示系统

#### Could Have（可以实现）- P2
- [C1] PWA支持（离线可用）
- [C2] 快捷键支持
- [C3] 无障碍适配（ARIA）
- [C4] 振动反馈（移动端）

#### Won't Have（本期不实现）- P3
- [W1] 云端同步（需后端改造）
- [W2] 多用户协作（权限系统）
- [W3] AI智能推荐（需API接入）

## 🔧 五、技术架构方案

### 5.1 Vercel微信访问修复方案

#### 方案概述
采用**组合方案**：优化Vercel配置 + Hash路由兜底

#### 实施步骤

**Step 1: 优化vercel.json配置**
```json
{
  "version": 2,
  "routes": [
    {
      "src": "/assets/(.*)",
      "headers": { 
        "Cache-Control": "public, max-age=31536000, immutable" 
      }
    },
    {
      "src": "/(.*)\\.(js|css|png|jpg|jpeg|gif|ico|svg|webp|woff|woff2|ttf|eot)",
      "headers": { "Cache-Control": "public, max-age=31536000" }
    },
    {
      "src": "/(.*)",
      "dest": "/index.html",
      "headers": {
        "X-Frame-Options": "SAMEORIGIN",
        "X-Content-Type-Options": "nosniff",
        "X-XSS-Protection": "1; mode=block",
        "Referrer-Policy": "strict-origin-when-cross-origin"
      }
    }
  ],
  "regions": ["hkg1", "sfo1"],
  "github": {
    "silent": true
  }
}
```

**Step 2: 路由模式改为HashRouter**
```jsx
// src/App.jsx
import { HashRouter as Router } from 'react-router-dom';

function App() {
  return (
    <Router>
      {/* 路由配置 */}
    </Router>
  );
}
```

// src/utils/wechatDetector.js

#### 风险控制
