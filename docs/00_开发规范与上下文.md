2. AI统一协作规范
2.1 AI交互原则
•定位清晰： 只做方案生成，不涉及交易环节
•合规优先： 严格遵循政府采购相关规定
•简洁高效： 提供最简化的解决方案
2.2 文案生成规范
2.2.1 术语规范
•严禁出现：购买、下单、支付、商城、购物车
•出现自动替换为：采购、方案、预算、物资
2.2.2 消费帮扶提示
统一使用标准文案：
"温馨提示：为完成年度消费帮扶任务，建议在食品类采购中优先选用832平台产品，便于集中完成全年指标。"
2.2.3 合规性提示
•人均预算超过500元时，自动添加提醒
•涉及财政拨款时，不生成特殊逻辑
2.3 AI生成逻辑
2.3.1 公文生成
•强制使用Few-Shot Prompting（少样本提示），严禁自由发挥
•如果API调用失败，展示本地预设的静态模板文本
•变量替换规则：
◦[节日名称] → 场景选择（春节/中秋/其他）
◦[金额] → 总预算 / 人数
◦[商品名称] → 品单列表循环输出
◦[15%以上] → 动态计算832商品占比
2.3.2 错误处理
•网络异常：显示"AI服务暂时不可用，请稍后重试"
•数据异常：显示"输入数据有误，请检查后重新提交"
•系统异常：显示"系统出现错误，请联系管理员"

4.3 API接口
4.3.1 商品相关接口
•GET /api/products - 获取商品列表
